# Claude Code 運用ガイド

## 変更履歴の記録

変更履歴は `docs/CHANGELOG.md` に蓄積する。  
作業完了後に「変更履歴を記録して」と指示されたら、ファイルの**先頭**（`---` の直後）に新しいエントリを追記すること。

### エントリ形式

```markdown
## YYYY-MM-DD

### <変更のタイトル>

**ブランチ:** `<branch-name>`

**概要:**  
<何を・なぜ変えたかを2〜4文で記述>

**変更ファイル:**
- `<path>` — <変更内容の一言説明>
```

### 記録のタイミング

- ユーザーから「変更履歴を記録して」「CHANGELOG を更新して」などの指示があったとき
- コミット・プッシュの後に記録を求められたとき

### 注意

- エントリは新しい順（先頭が最新）に並べる
- ブランチ名・変更ファイル・概要は省略しない
- 変更が複数のフィーチャーにまたがる場合は `###` を複数書いてよい
