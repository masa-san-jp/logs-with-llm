# チャットの✍️リアクション1つで、タスクが自動でシートに記録される仕組みを作った話

## 情報が増えすぎて、タスクが消えていく

チャットツールを使い込んでいくと、あるいつの間にか起きていることがある。スペース、グループチャット、個別チャット、そしてメール。情報の置き場所が増えるほど、「あの話、どこでしてたっけ？」が増えていく。

ある部門長は、こう語る。

> 「チャットが無限に増えていくんです。どこで何の話をしていたか覚えられないし、タスクが抜けてしまう」

タスク管理ツールへの転記を徹底しようとしても、それ自体がひと手間だ。チャットを見ながら別のツールを開いて、コピーして、貼り付けて——その手間が積み重なると、いつか省略される。

## 解決策は「絵文字リアクション1つ」だった

この課題に対して生まれたのが、[GWS-emoji-2-task-sheet](https://github.com/masa-san-jp/GWS-emoji-2-task-sheet) という仕組みだ。

Googleチャットのメッセージに ✍️ のリアクションをつけるだけで、そのメッセージが自動的にスプレッドシートへ記録される。

なぜ絵文字リアクションだったのか。

> 「今対応するか、後で対応するかを、1アクションで切り分けたかった」

そしてもう一つの理由がある。リアクションはチャットを送った相手にも見える。つまり ✍️ をつける行為は、「タスクとして受け取りました」という意思表示にもなる。タスク登録と返答が、1アクションで同時に完結する。

## 技術的なポイント：2段階構成が必要だった理由

フローの構築にはGoogle Workspace Studioを使っている。「絵文字リアクションが追加されたとき」をトリガーに、スプレッドシートへ行を追加するフローをノーコードで作ることができる。

しかしここに制約があった。

> 「Workspace Studioの制約で、共有スプレッドシートには直接書き込めなかった」

そこで一度、自分だけがアクセスできる個人用スプレッドシートに書き込み、そこからGoogle Apps Script（GAS）が定期的に共有スプレッドシートへ転記する、という2段階の構成になっている。

```
✍️リアクション
　↓
Workspace Studio
　↓
個人用スプレッドシート（中継）
　↓
GAS（タイマー実行）
　↓
共有スプレッドシート
```

GASのコードや設定方法の詳細は[GitHubリポジトリ](https://github.com/masa-san-jp/GWS-emoji-2-task-sheet)を参照してほしい。

## 運用の型：タスク形式で返信してからリアクションする

ツールを使いこなすうえで、もう一つ重要な工夫がある。チャットの内容が必ずしもタスク形式になっているとは限らない。たとえば相手からこんなメッセージが来たとする。

> 「できたら今週中に！🙏」

このままスプレッドシートに記録しても、「何を、誰に、いつまでに」が不明瞭だ。そこでタスク形式の返信をしてから、そこにリアクションをつけるようにしている。

```
（相手）できたら今週中に！🙏
└（私）4/18までに○○さんに、○○のレビュー結果を伝える ✍️
```

リアクションをつけるのは、あくまで自分の返信。これにより、スプレッドシートには「誰が、何を、いつまでに」という構造化されたタスクが記録される。

## 想定外の使われ方が生まれた

運用を始めたあと、予想していなかった使い方がチーム内で自然発生した。

ツールを自分で設定・運用することが難しいメンバーが、定時MTGの中で自発的にこんな使い方を発明したのだ。

> 「@私　これ✍️でお願いします！」

部門長をタスク登録の代理人として使う形だ。誰かが頼めば、部門長がリアクションをつけてタスク化する。ツールを直接使えなくてもワークフローに乗れる、予想外の間口の広さだった。

## 公開した理由

このスクリプトはGitHubで公開されている。その理由を聞くと、こんな答えが返ってきた。

> 「自分が困っているということは、世の中の誰かも困っているということ。私が今便利に仕事できているのも、誰かが解決策を公開してくれていたおかげ。だから公開するのは当然のことだと思っている」

課題解決の循環を社会に還元する、という思想だ。
