# Specification‑First Design Across Dialogue, Mentoring, and Crisis Systems  

The week’s work compared four independent design efforts that each enforce deterministic behavior through explicit specifications, modular skill runtimes, or structured prompts. The comparison highlights how state‑driven protocols, skill‑layered AI mentors, and procedural chatbot prompts converge on a common strategy for reducing model‑driven ambiguity.

## Complex Interrogation Protocol: Phase Structure and Guardrails  
The “執拗追及型コンプレックス深掘り対話プロトック仕様書” (v1.0, 2026‑05‑30) defines eleven sequential phases, from **Opening Declaration** to **Core Statement and Actionization**. Each phase specifies an initial question, follow‑up questions, and an attainment condition; a minimum of three repeats per theme is enforced. Defense‑reaction categories (e.g., “generalization”, “intellectualization”) are listed with corresponding intervention prompts, and a depth meter grades responses from Level 0 (avoidance) to Level 7 (actionization). Termination occurs only when four core components—true desire, shame perception, value condition, and concrete 24‑hour action—are present, or when a disallowed state (self‑destructive impulse) is detected. This protocol supplies a deterministic state diagram that can be encoded in a stateful AI agent.

## Personal AI Mentor Kit: Skill Runtime Architecture on Claude Code  
Meeting notes from 2026‑05‑30 record a shift from a RAG‑style design to a **Skill Runtime** model hosted on Claude Code. Five MVP skills (idea_review, writing_review, decision_review, contradiction_finder, next_action_planner) are defined by purpose, trigger, inputs, procedure, rubric, output format, and failure modes. Knowledge cards are organized into ten types (worldview, principle, decision rule, etc.) and layered into five strata (worldview → interaction). All assets reside in Markdown/YAML files, allowing the file system to act as the mentor’s operating system. The design explicitly excludes copyrighted text and personal logs, focusing on “judgment style” rather than impersonation. The implication is that skill‑based execution can produce concrete, actionable advice without relying on document retrieval.

## Crisis‑Response Chatbot: System Prompt Objectives and Interaction Flow  
The system prompt dated 2026‑06‑01 outlines nine ordered objectives for an internal emergency chatbot, beginning with “confirm life and body safety” and ending with “decide next update time.” The prompt mandates a non‑decision‑maker role, strict separation of facts, unverified information, and speculation, and a three‑question maximum per reply. A fixed initial question set (life safety, damage spread, safe location) triggers a deterministic question sequence of up to twelve steps for unknown incidents. Templates for first‑report and record creation are provided, and the bot must refuse actions outside its scope (e.g., legal judgments, credential sharing). This structure enforces deterministic guidance while leaving content generation to the language model.

## Business Office Skills Framework: Repository Layout and Enforcement Rules  
The “business‑office‑skills‑framework” adapts data‑science design philosophy to office automation. The repository contains a thin router (`AGENTS.md`), skill definitions under `skills/*/SKILL.md`, organization‑specific documentation under `docs/*`, and prompt files under `prompts/*`. Absolute rules in `AGENTS.md` prohibit overwriting originals, sending confidential data, and asserting uncertain facts. Each skill file encodes procedural checks (e.g., email‑drafting enforces subject line, greeting, and attachment verification). The framework relies on CI‑style checklists (`checklists/`) for mechanical compliance because traditional CI pipelines are unavailable for pure document work. The design demonstrates that specification‑driven skill modules can enforce business policies without runtime model supervision.

**Synthesis** – All four artifacts employ a specification‑first approach: the dialogue protocol encodes phase transitions, the mentor kit isolates judgment into discrete skills, the crisis chatbot embeds nine immutable objectives, and the office framework structures tasks through enforced checklists. The convergence suggests that deterministic scaffolding, rather than model scaling, is the primary lever for reliable AI‑augmented processes. Future work will explore cross‑domain integration of these scaffolds, such as applying the skill‑runtime model to the complex‑interrogation phases or extending the crisis‑response prompt to incorporate the office checklist taxonomy.
