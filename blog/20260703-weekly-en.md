# Separate Claude Code Accounts via Config Directory

The week’s investigation confirmed that distinct Claude Code sessions can coexist on a single Linux host by delegating each account to its own `CLAUDE_CONFIG_DIR`. The approach eliminates token overwriting and preserves background processes that depend on a specific credential set.

## Authentication File Location and Priority  

Claude Code CLI stores OAuth tokens in `.credentials.json` under a configuration directory. By default Linux uses `~/.claude/.credentials.json` (mode 0600); Windows uses `%USERPROFILE%\.claude\.credentials.json`; macOS stores the token in Keychain. The CLI resolves authentication in the following order: `ANTHROPIC_API_KEY`, `ANTHROPIC_AUTH_TOKEN`, `.credentials.json` in `CLAUDE_CONFIG_DIR`, then the default `.credentials.json`. Consequently, an environment‑level `ANTHROPIC_API_KEY` overrides file‑based OAuth, requiring explicit `unset` when switching accounts.

## Environment Variable `CLAUDE_CONFIG_DIR` for Account Isolation  

Setting `CLAUDE_CONFIG_DIR` redirects the CLI to a user‑specified directory, e.g., `$HOME/.claude-alt`. Each directory contains its own `.credentials.json` and `settings.json`, ensuring that login, logout, and status operations affect only the targeted account. The variable is evaluated before the default path, so the CLI never falls back to the primary configuration while the variable is defined.

## Wrapper Script Implementation (`claude-alt`)  

A minimal Bash wrapper enables one‑command access to the secondary account:

```bash
#!/bin/bash
export CLAUDE_CONFIG_DIR="$HOME/.claude-alt"
exec claude "$@"
```

After `chmod +x ~/.local/bin/claude-alt`, the script is invoked as `claude-alt auth login` to initialize the alternate OAuth flow, and subsequently as `claude-alt` for all CLI interactions. Parallel usage (`claude` vs. `claude-alt`) confirms that both `.credentials.json` files remain independent, as verified by `claude auth status` and `claude-alt auth status`.

## Operational Caveats and Platform Differences  

The default account must not be logged out with `claude auth logout` when persistent agents rely on its token; logout should be performed only through the wrapper to avoid unintended credential loss. Unsetting `ANTHROPIC_API_KEY` is mandatory for OAuth account switches because the key supersedes file‑based tokens. On macOS, Keychain storage cannot be partitioned by directory, so multi‑account isolation must rely on distinct API keys or external helpers. The same directory‑splitting pattern applies across subscription tiers (Pro, Max, Team, Enterprise) and respects organization‑level SSO policies, provided that `settings.json` is also kept per‑directory.

**Synthesis** – Isolating CLI credentials via `CLAUDE_CONFIG_DIR` demonstrates a scalable method for multi‑tenant command‑line tools. The pattern decouples authentication state from global environment, reduces risk of inadvertent logout, and supports concurrent workflows without version or binary changes. Future extensions could formalize wrapper generation, integrate automatic `ANTHROPIC_API_KEY` cleanup, and expose similar isolation mechanisms for other token‑based CLIs.
