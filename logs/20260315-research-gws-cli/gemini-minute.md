# 議事録：組織の意思決定プロセス可視化・AI化プロジェクト キックオフ

## 1. プロジェクトの目的
組織全体の活動の中で「暗黙的に繰り返されている個人の感覚や経験に依拠した意思決定・業務プロセス」を可視化し、業務の自動化およびAI導入を強烈に促進する。

## 2. コアとなる方針（パラダイムシフト）
* **【脱・単なるグラフ化】** BIツールによる定量的な可視化（誰が何回発言したか、会議時間など）ではなく、LLMによる**「意味的・構造的な可視化（誰のどんな発言で仕事が進んだか／止まったか）」**を重視する。
* **【暗黙知の言語化】** Google Workspace上の生データ（特にチャットやドキュメントのコメント）をLLMに解析させ、人間が気づいていない「組織の癖」「ボトルネック」「暗黙のルール」を抽出する。

## 3. アプローチとデータソース
Google Workspace CLIを活用し、以下の優先順位でデータを抽出し、LLMパイプラインに流し込む。

* **Phase 1（直近のスコープ）**: 特定のGoogle Chatスペースのログ取得。
  * 狙い: 意思決定のプロセス、ネクストアクション、タスク停滞の要因（承認待ち等）の抽出。
* **Phase 2（将来的な拡張）**: Google Docs/Sheetsのコメント・修正履歴の取得。
  * 狙い: 暗黙の品質基準や判断軸の可視化。

## 4. システムアーキテクチャ・開発環境
* **開発体制**: GitHubを起点とし、ローカルIDE（VSCode + Claude Code / Antigravity）およびクラウド（Claude Code Cowork / GitHub Copilot）を活用したAIネイティブな開発フロー。
* **技術スタック（想定）**: Python 3.11+, Pydantic (データスキーマ定義), LLM SDK (Anthropic等)。
* **設計ドキュメント**: `design_doc.md` として、入力ログと出力インサイト（`InsightReport`）のPydanticスキーマ、および推奨ディレクトリ構成を定義済み。

## 5. 決定事項とネクストアクション
1. **[決定]** プライバシーと心理的安全性を担保するため、抽出するログデータは個人名（PII）をマスキングする前処理（`Member_A`等への変換）を必ず実装する。
2. **[決定]** 開発初期は、いきなり本番のCLIデータを叩くのではなく、ダミーデータ（JSON）を用いたローカルテストから着手する。
3. **[Next Action]** 開発担当（エージェント含む）は、リポジトリに配置された `design_doc.md` を読み込み、`schemas/` ディレクトリ配下のPydanticモデル定義、およびディレクトリ構成の作成に着手する。
