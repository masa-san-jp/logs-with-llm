# **Google Workspace 活用プロジェクト提案書**

---

## **プロジェクト概要**

### **タイトル**:
- 組織の意思決定と業務プロセスの可視化と改善

### **目的**:
1. **暗黙知の明示化:**
   - 現在、人間同士のコミュニケーションによって非体系的に進められている意思決定プロセスや業務プロセスをデータ化。
   - チームやプロジェクト単位での「見える化」を目指す。
   
2. **AI（LLM）の活用:**
   - データを大規模言語モデル（LLM）に提供し、インサイト（洞察）を発見。
   - 次のアクションを提案するデータ駆動型組織を構築。

3. **自動化と効率化:**
   - 業務フローや意思決定のボトルネックを特定・自動化し、効率を改善。

## **期待されるバリュー**

1. **透明性の向上**
   - 誰が、いつ、どこで、どのような意思決定を行ったかをログとして記録。
   - 従来の個人の記憶や依拠に頼らず、プロセスの再現性を確保。

2. **プロセス改善**
   - データから頻繁に発生するタッチポイント（例：会議、メール、タスク管理）を特定し、業務フロー全体を改善。

   - LLMを通じて、これまで気づかなかった行動パターンや問題を自動的に発見。
   - 重要なトピックやネクストアクションに対する意思決定支援。

4. **組織間連携の強化**
   - 異なるチーム間のコミュニケーションと作業を統合し、シームレスな業務環境を提供。

## **使用ツールと具体的な方法**

### **ツール: [Google Workspace CLI](https://github.com/googleworkspace/cli)**

#### 概要:
- Google Workspace CLI（以下「gws CLI」）は、Google Workspace製品（Drive、Gmail、Calendar、Chatなど）を操作するための動的・構造化CLIツール。
- REST APIを基にコマンドを構成し、データ取得や操作をシンプルに実行可能（スクリプトやインテグレーション対応）。

#### **CLIサポートされているアクション例:**
1. **Google Calendar**:
   - カレンダーイベント（例：会議）のリスト化。会議の時間帯や頻度を分析。
   ```bash
   gws calendar events list --params '{"timeMin": "2026-03-01T00:00:00Z"}' --fields "items(summary,start,end,attendees)"
   ```

2. **Google Chat**:
   - 会話ログを収集し、頻出トピックを分析。コミュニケーションの流れを可視化。
   ```bash
   gws chat spaces messages list --params '{"parent": "spaces/<space-id>"}'
   ```

3. **Google Drive**:
   - ドキュメントの履歴・変更記録を収集。重要なリソースの影響力を測定。
   ```bash
   gws drive files list --params '{"q": "modifiedTime >= '2026-03-01'"}' --fields "files(name, owners, webViewLink)"
   ```

#### **期待されるデータ収集・処理フロー:**
1. **データ取得**:
   gws CLIにより各データソースから構造化ログ(JSON形式)を取得。
2. **データ整備**:
   データクレンジング（例：重複排除、プライバシー保護用匿名化）。
3. **分析システムに提供**:
   LLM（例：ChatGPT）のコンテキストで解析に提供。
4. **インサイト作成**:
   分析結果を視覚化ツール（例：Looker、Tableau）で可視化。

## **デモンストレーションシナリオ**

デモを行う際には以下のシナリオを提示できます：
1. **カレンダーイベントを分析する流れ:**
   - CLIで特定期間のイベントを収集し、参加者やテーマの傾向をビジュアライズ。
2. **チャットログのキーワードクラスタリング:**
   - 特定スペースの議論テーマを解析、主要な焦点を浮き彫りに。
3. **ドライブのファイル関係性の解析:**
   - プロジェクト依存ファイル構造を特定し、課題を特定。
