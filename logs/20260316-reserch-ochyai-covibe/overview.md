co-vibe プロジェクト概要
概要
co-vibe は、Anthropic・OpenAI・Groq・Ollamaの4プロバイダを横断するマルチプロバイダAIコーディングエージェント。Pure Python、外部依存ゼロのシングルファイル実装。 ￼
タスクの複雑度を自動判定して最適なモデルに振り分ける「3段オーケストレーション」が中核機能。研究開発を目的とした透明性・デバッグ性重視の設計。

アーキテクチャ

User ←→ TUI
         ↓
       Agent（3段タスク分類器）
         ↓
   MultiProviderClient（ヘルスチェック・自動フェイルオーバー）
         ↓
  Anthropic / OpenAI / Groq / Ollama
         ↓
   ToolRegistry（Bash, ReadFile, WriteFile, Edit, Glob, Grep, WebFetch, WebSearch + 10以上）
   SubAgent / DeepResearch / MCP / Notebook / Parallel


主要機能



|機能           |内容                                                              |
|-------------|----------------------------------------------------------------|
|3段ルーティング     |`simple → fast`, `normal → balanced`, `complex → strong` の自動振り分け|
|Deep Research|タスク分解 → 並列Web検索 → 統合の多段階調査                                      |
|マルチエージェント    |ワークスティーリングスレッドプールによる並列サブエージェント実行                                |
|セッション永続化     |コンテキスト自動圧縮付きの保存・復元                                              |
|MCPサポート      |Model Context Protocolによる外部ツール連携                                |
|コスト追跡        |ターンごとのトークン使用量・推定コスト表示                                           |
|Thinking表示   |`<think>` ブロックによるモデル推論のリアルタイム表示                                 |

ユースケース
	1.	ターミナルベースのAIペアプログラミング
自然言語でコーディング指示 → ファイル生成・編集・コマンド実行を対話的に実施
	2.	Deep Webリサーチ
複雑な調査タスクを自動でサブクエリに分解し、並列検索・統合レポート生成
	3.	マルチエージェント並列開発
大規模タスクを複数のサブエージェントに分散実行
	4.	コスト最適化コーディング
cheap / fast / strong の戦略指定で用途に応じたコスト・速度トレードオフ制御
	5.	プライベート・オフライン利用
Ollamaを使ったローカルモデルでAPIキー不要・完全オフライン運用
	6.	AI研究・実験基盤
--debug でAPIコール・ツール実行・判断過程をすべてトレース可能なため、エージェント研究のプラットフォームとして利用
