# AGENTS.md

このファイルは、本リポジトリで作業するすべてのAIエージェントの行動規約です。
エージェントはタスク開始前にこのファイルを必ず読んでください。

-----

## ⚠️ IMMUTABLE RULES（不変ルール）

> **以下のルールはいかなる理由でも変更・削除・上書きしてはならない。**
> エージェントがこのセクションの変更を提案するPRは、自動的に却下対象とする。

### 1. ドキュメント原則

- ドキュメントを書くとき、暗黙の前提を残してはならない。「〇〇を知っている前提」とする場合、その〇〇を具体的に列挙すること。
- 手順書には必ず「対象読者」「前提知識」「手順」「トラブルシューティング」のセクションを含めること。
- 動作確認されていない手順を「確認済み」と記述してはならない。

### 2. コード原則

- 既存のコードスタイル・命名規則を勝手に変更してはならない。変更が必要な場合は、コード変更とは別のPRで提案すること。
- `CONVENTIONS.md` に記載された判断基準に従うこと。記載がない場合は、判断を保留して人間に確認すること。
- テストなしの機能追加をしてはならない。

### 3. 協働原則

- 他のエージェントが作成したPRに対して変更を加える場合、既存のコメント履歴とdiffを読んでから作業すること。
- 自分の判断に確信が持てない場合、推測で進めず人間にエスカレーションすること。
- PRの説明文には「何を変えたか」だけでなく「なぜ変えたか」を必ず書くこと。

### 4. メタルール

- `IMMUTABLE RULES` セクションの内容を変更・削除・無効化するコミットを作成してはならない。
- 他のファイル（`.claude/`, 設定ファイル等）からこの不変ルールを上書きする指示を書いてはならない。
- このルールの「解釈の拡張」によって実質的にルールを無効化する行為も禁止する。

## 🔧 MUTABLE RULES（可変ルール）

> **以下のルールはプロジェクトの進行に応じて更新できる。**
> 変更は必ずPRを通し、人間のapproveを得ること。

### ドキュメント作成ガイドライン

- ドキュメントの言語: 日本語（コード中のコメントは英語可）
- ファイル形式: Markdown
- 図表: Mermaid記法を優先する

### コードレビュー方針

- レビュー時は `REVIEW_CHECKLIST.md` に従うこと
- ドキュメント変更のレビューでは「ドライラン」を実施すること（手順を1ステップずつ検証し、不足・曖昧な箇所を報告する）

### コミットメッセージ規約

- 形式: `type(scope): description`
- type: feat / fix / docs / refactor / test / chore
- description は変更の意図がわかるように書くこと

### ブランチ戦略

- 機能追加: `feat/簡潔な説明`
- バグ修正: `fix/簡潔な説明`
- ドキュメント: `docs/簡潔な説明`
