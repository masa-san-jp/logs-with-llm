# REVIEW_CHECKLIST.md

PRレビュー時にエージェントおよび人間が使用するチェックリストです。

> **運用ルール**: このチェックリストはプロジェクトの進行に応じて項目を追加・修正できる。
> ただし `AGENTS.md` の IMMUTABLE RULES に由来する項目（🔒マーク）は削除・無効化してはならない。

-----

## ドキュメント変更のレビュー

### ドライラン（必須）

手順書の変更が含まれるPRでは、以下を実施する:

1. 手順を1ステップずつ検証する
1. 各ステップで以下を報告する:
- 書かれていない暗黙の前提はないか
- 判断が必要な曖昧な箇所はないか
- バージョンや環境に依存する箇所はないか
1. 報告フォーマット:

```
## ドライラン結果

### ステップ N: （ステップ名）
- ✅ 問題なし / ⚠️ 指摘あり
- 指摘内容: （具体的に記述）
- 推奨修正: （あれば）
```

### 破壊テスト

ドライランに加えて、以下の観点で脆弱な箇所を洗い出す:

- この手順に従う人が間違えやすい箇所はどこか
- OS・環境の違いでコケそうな箇所はどこか
- 権限やネットワークの前提が暗黙になっていないか

-----

## コード変更のレビュー

### 基本チェック

- [ ] 🔒 `CONVENTIONS.md` の命名規則・スタイルに準拠しているか
- [ ] 🔒 テストが追加・更新されているか
- [ ] 🔒 PRの説明に「何を」「なぜ」変えたか書かれているか
- [ ] 変更の影響範囲は適切か（関係ないファイルを巻き込んでいないか）
- [ ] エラーハンドリングは適切か
- [ ] パフォーマンスへの影響を考慮しているか

### セキュリティチェック

- [ ] 機密情報（API キー、パスワード等）がハードコードされていないか
- [ ] ユーザー入力のバリデーションは適切か
- [ ] 依存パッケージに既知の脆弱性はないか

### ドキュメント整合性

- [ ] 🔒 コードの変更に対応するドキュメント更新がされているか
- [ ] README や設定例が古くなっていないか

## レビュー結果のアクション

レビューで発見した知見の反映先:

|発見内容            |反映先                            |
|----------------|-------------------------------|
|今後も当てはまるルール・判断基準|`CONVENTIONS.md` に追記           |
|繰り返し発生する間違い     |`CONVENTIONS.md` の「既知の罠」に追記    |
|レビュー観点の追加が必要    |このファイルに項目追加                    |
|行動規約レベルの変更      |`AGENTS.md` の MUTABLE RULES を更新|
