# 議事録：生成AI利用規程（就業規則別紙）策定に向けた調査報告と方針検討

**作成日:** 2026年4月3日
**テーマ:** 業務における生成AIの安全な利用（特にローカルファイル・ターミナル操作）に関する社内規程策定
**目的:** 調査内容を共有し、就業規則別紙および運用ガイドラインの正式な策定に向けた意思決定を行う

---

## 1. 背景・目的
生成AI（AIエージェント等）の業務利用による生産性向上が見込まれる一方、意図せぬローカルファイルの改ざん・消失、および不正プログラム実行のリスクが顕在化している。
特に、ビジネスサイドを含む全社員が利用することを想定し、自社のGoogle Workspace環境に特化した安全基準とルールを「就業規則 別紙（生成AI利用規程）」として定める必要がある。

## 2. 他社事例および標準ガイドラインの動向
* **業界標準の活用:** 日本ディープラーニング協会（JDLA）等のひな形をベースにカスタマイズする手法が一般的。
* **行政・教育機関の指針:** 東京都庁や各大学でも「機密情報の入力制限」や「生成物のファクトチェック義務化」を明文化しており、本規程案もこれらの水準に準拠して作成している。

## 3. 識別された主要リスク（Google Workspace環境特有の課題）
当社の環境（データは原則クラウド上）において、最も警戒すべきは **「クラウド同期アプリ（Google ドライブ パソコン版など）を通じたデータの連鎖的な消失・破壊」** である。

* **シナリオ:** AIがローカルの同期フォルダ上で誤って全ファイル削除等の破壊的コマンドを実行した場合、その変更が即座にクラウドへ同期され、全社の共有データが消失するリスクがある。
* **ビジネスサイド特有のリスク:** AIが提案したターミナルコマンドやプログラムの意味を理解せずに「盲目的に実行（コピペ等）」してしまうことによる事故・マルウェア感染。

## 4. 規程への組み込み事項（ルールによる防衛）
上記の事態を防ぐため、以下の項目を規程（特則）として明記する。

| 項目 | 規程・ルールの内容 |
| :--- | :--- |
| **作業環境の分離** | 同期フォルダでの直接操作を禁止。必ずローカル（デスクトップ等）に「AI作業用フォルダ」を作成し、ファイルをコピーしてから処理・検証を行う。 |
| **確認の義務化** | AIによる自律的な連続実行（フルオート）を禁止。人間による目視確認と手動実行（Human-in-the-Loop）を必須とする。 |
| **理解なき実行の禁止** | 内容を理解・説明できないプログラムやコマンドを、AIの提案のみを理由に実行してはならない。 |
| **権限の制限** | AIツールやターミナルの「管理者権限（sudo, Administrator等）」での実行を禁止。 |

## 5. 技術的対策と運用・教育（システムによる防衛）
規程と併せて、以下のシステム制御およびリテラシー教育を実施する。

1. **システム制御:**
   * 一般社員PCにおけるローカル管理者権限の原則剥奪。
   * EDR（エンドポイントセキュリティ）による不審なスクリプト実行の検知・遮断。
2. **周知・教育（マニュアル化）:**
   * 「危険コマンド一覧（`rm -rf`, `sudo`, `chmod 777`, `curl | bash` 等）」をガイドラインとして策定・配布。
   * 「危険な文字列が含まれていた場合は実行せずにIT部門へ相談する」フローの確立。
   * 万が一の際のGoogle Workspaceのバージョン履歴からの復元手順の周知。

---

## 6. 今後の意思決定・アクションアイテム（Next Steps）
経営陣・法務部門にて以下の事項について確認・承認を求める。

- [ ] **就業規則の一部改訂:** 提示された「生成AI利用規程（案）」を就業規則の別紙として正式採用することの承認。（必要に応じ顧問社労士・弁護士によるレビューを実施）
- [ ] **対象ツールの選定:** 全社で利用を許可する法人向け生成AIサービス（オプトアウト済み環境）の選定と予算化。
- [ ] **IT部門の体制構築:** ビジネスサイドから「複雑な自動化処理」や「危険コマンドの判定」について相談を受けた際のサポート体制の構築。
- [ ] **全社展開のスケジュール:** 従業員向け説明会の実施、および「安全利用マニュアル（危険コマンドリスト含む）」の配布時期の決定。
