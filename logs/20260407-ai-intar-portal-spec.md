# 社内AIポータルサイト 全体設計書

> **設計思想**: このポータルは「使いながら学ぶ」構造を持つ。Claude CodeのCLAUDE.mdやSkillディレクトリのように、上位概念（なぜAIを使うか）→中位概念（何をどう使うか）→下位概念（具体的なスキル・エージェント）が自然に理解できる階層構造を持つ。

## CLAUDE.md（社内AI共通コンテキスト）の役割

`/prompt-library/CLAUDE.md` は、社内のAI利用における「共通前提」を定義するファイルです。  
Claude CodeのCLAUDE.mdと同じ考え方で、全てのAIプロンプトの「暗黙知」を明文化します。

```markdown
# 社内AI共通コンテキスト（CLAUDE.md）

## 会社・業種情報
## 利用シーン別の注意事項
## 社内専門用語・略語辞書
## 絶対に入力してはいけない情報（機密分類）
## 出力品質の基準（トーン・形式・言語）
## エスカレーションルール（AIで対応不可な場合）
```

## 情報フロー設計（付加価値の循環）

```
社員がツールを使う
       ↓
Gem・プロンプトライブラリを参照
       ↓
業務成果（時間短縮・品質向上）が出る
       ↓
事例として /use-cases/ に投稿
       ↓
ROIが可視化される（roi-tracker.md）
       ↓
新しいGem・エージェントに昇華される
       ↓
全社員が恩恵を受ける（ループ）
```

## フェーズ別展開ロードマップ

### Phase 1（立ち上げ：1〜2ヶ月）
- [ ] /governance/ 整備（規定・ガイドライン・ツール一覧）
- [ ] /learning/ 整備（既存研修動画のリンク＋要点）

- [ ] CLAUDE.md（社内共通コンテキスト）の初版作成
- [ ] /use-cases/ 開設（事例投稿の促進）

### Phase 3（自律化：5〜6ヶ月以降）
- [ ] /agents/ 整備（審査済みエージェントの本番公開）
- [ ] ROIトラッカー稼働（工数削減の可視化）
- [ ] 部門別AI担当者（AI Champion）制度の立ち上げ
