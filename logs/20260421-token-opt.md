# 議事録: Agent-Workplace トークン最適化設計

**日時**: 2026-04-21  
**参加者**: プロダクトオーナー  
**テーマ**: Claude Code スキルのトークン節約テクニックを Agent-Workplace プロジェクトへ適用する設計検討

-----

## 背景・課題

[DevelopersIO記事](https://dev.classmethod.jp/articles/claude-code-skill-markdown-token-saving-tips/)で紹介されたトークン節約テクニックを Agent-Workplace プロジェクトへ適用したい。  
特に「過去進捗をmdで管理しているため、将来的にトークン消費が激しくなる」ことが懸念事項として挙げられた。

-----

## 参照記事の6Tips（前提知識）

|Tips|内容                                                    |
|----|------------------------------------------------------|
|1   |既存Lintツール（textlint等）を使い、フォーマットチェックをClaudeから排除         |
|2   |正規表現・キーワードマッチで判定できるチェックをPythonスクリプトに切り出す              |
|3   |参照ファイルをPythonで前処理し、該当行のみClaudeに渡す                     |
|4   |ファイルの必要なセクションだけをPythonで抽出してClaudeに渡す                  |
|5   |URL固定の外部データはMCP/WebFetchではなく `urllib.request` で直接取得   |
|6   |Pythonで検知 → `needs_llm_review` フラグ → 該当箇所のみClaudeが文脈確認|

**根本思想**: 「Claudeには、Claudeにしかできないことだけをやらせる」

## 議論の経緯

### Phase 1: 基本的な組み込み方針

AGENTS.md / SKILL.md ベースのプロジェクトに対し、以下の組み込み方針を策定した。

- `scripts/precheck.py` を新規追加し、決定論的チェック（必須セクション確認・NG表現検出・見出し階層チェック等）をPythonに移行
- `scripts/fetch_section.py` を新規追加し、外部URL取得をMCP/WebFetch代替としてPythonで処理
- AGENTS.md の MUTABLE RULES に「Claudeにしかできないことだけをやらせる」原則を明文化
- SKILL.md は「Python前処理 → Claude判断」の2ステップ構造をテンプレート化

### Phase 2: 進捗mdファイルのトークン問題

過去進捗をmdで管理しているため、運用期間が長くなるにつれてエージェントの読み込み量が増加し続けるという問題を検討した。

**検討した解決策**: 3層メモリ構造

```
層1: _summary.md（圧縮済み過去・最大150行）   ← エージェントが読む
層2: _recent.md（直近14日・最大100行）        ← エージェントが読む
層3: archive/（月別フルログ）                 ← 原則エージェントは読まない
```

`compress_logs.py` を月次実行することで、何ヶ月運用してもエージェントの読み込み量を常に250行固定に維持できる設計とした。

### Phase 3: ファイル形式の再検討（決定事項）

「トークン節約という観点でmdが最適か？」という問いから、ファイル形式の比較検討を実施した。

|形式      |トークン効率|可読性|エージェント処理|
|--------|------|---|--------|
|Markdown|△     |◎  |△       |
|JSON    |◎     |△  |◎       |
|JSONL   |◎     |△  |◎       |
|YAML    |○     |○  |○       |

**結論・決定事項**: エージェントチームとして最適化するため、**エージェントが扱うファイルはすべてJSON/JSONL形式に統一する**。人間が読みたい場合はChatGPT等の外部ツールで都度変換する運用とする。

## 最終アーキテクチャ（決定）

### ディレクトリ構成

```
Agent-team/
├── spec.json                      # 仕様（旧spec.md）
├── agents/
│   └── {agent-name}/
│       ├── config.json            # エージェント定義
│       ├── input/*.json
│       └── output/*.json
├── logs/
│   ├── current.jsonl              # 当月ログ（追記専用）
│   ├── _recent.jsonl              # 直近14日（自動生成）
│   ├── _summary.jsonl             # 月次集計サマリー（自動生成）
│   └── archive/
│       ├── 2026-01.jsonl
│       └── ...
├── tasks/
│   ├── backlog.json               # 全タスク管理
│   └── active.json                # 進行中のみ（自動生成）
└── skills/
    ├── compress_logs.py           # 月次ログ圧縮
    ├── sync_tasks.py              # アクティブタスク同期
    └── to_markdown.py             # 人間向け変換（ChatGPT等に渡す用）
```

### スキーマ定義

**`logs/current.jsonl`（1行=1エントリ）**

```json
{
  "ts":        "YYYY-MM-DD",
  "agent":     "エージェント名",
  "task":      "タスク名",
  "status":    "done | in_progress | blocked | cancelled",
  "summary":   "1行サマリー",
  "issues":    ["課題1"],
  "decisions": ["決定事項1"]
}
```

**`tasks/backlog.json`**

```json
{
  "updated": "YYYY-MM-DD",
  "tasks": [
    {
      "id": "T001",
      "title": "タスク名",
      "status": "done | in_progress | blocked | cancelled",
      "agent": "担当エージェント名",
      "priority": 1,
      "depends_on": [],
      "notes": ""
    }
  ]
}
```

### 運用フロー

```
【日次】
  python skills/compress_logs.py --recent-only  # _recent.jsonl 更新
  python skills/sync_tasks.py                   # active.json 更新

【月次（月初）】
  python skills/compress_logs.py                # アーカイブ＋SUMMARY更新

【人間がレビューしたい時】
  python skills/to_markdown.py all              # MD変換 → ChatGPT等に渡す
```

### エージェントの参照ルール

|用途      |読むファイル                     |最大量   |
|--------|---------------------------|------|
|進行中タスク確認|`tasks/active.json`        |件数による |
|直近ログ確認  |`logs/_recent.jsonl`       |直近14日分|
|過去サマリー  |`logs/_summary.jsonl`      |月次集計のみ|
|仕様確認    |`spec.json`                |固定    |
|エージェント設定|`agents/{name}/config.json`|固定    |

`logs/archive/` の一括読み込みは禁止。特定日付の詳細調査が必要な場合のみ参照する。

## 期待効果

|参照対象  |MD運用（6ヶ月後）|JSON運用              |削減率     |
|------|----------|--------------------|--------|
|ログ参照  |~1,200行   |~80行（SUMMARY+RECENT）|**93%** |
|タスク確認 |backlog全件 |active.jsonのみ       |~70%    |
|仕様参照  |spec.md全文 |spec.json（構造化）      |~30%    |
|**合計**|—         |—                   |**約85%**|
