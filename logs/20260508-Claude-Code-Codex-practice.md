# Claude Code と Codex の比較検証とベストプラクティス

## 目的

Claude Code と Codex の違いを、構造・設計・思想・得意不得意の観点から整理する。  
その上で、私自身を含めた役割分担を明確にし、実務で無理なく回るベストプラクティスを定義する。

## 要約

- Claude Code は、ローカル端末で人と密に協働する「近接型」の開発エージェント。
- Codex は、タスクを切り出して並列・非同期に進める「委譲型」の開発エージェント。
- どちらが優れているかではなく、開発工程のどこで使うと強いかが違う。
- 私は意思決定者であり、仕様・優先順位・採否を決める役割を持つ。
- Claude 系は「考える・詰める・仕上げる」に強く、Codex は「任せる・並列化する・検証する」に強い。

## 1. Claude Code と Codex の比較

### 1-1. 一言でいうと

- Claude Code: 今いる端末とコードベースの中で、一緒に考えながら進める相棒
- Codex: 明確な仕事を切り出して、別働隊として進める実行者

### 1-2. 構造の違い

| 観点 | Claude Code | Codex |
|---|---|---|
| 主戦場 | ローカル端末、現在の作業環境 | クラウド/隔離環境、並列タスク環境 |
| 基本モード | 対話型、逐次、伴走型 | 非同期、並列、委譲型 |
| 強みの出る単位 | その場の編集、調査、修正 | まとまったタスク、反復処理、複数候補試行 |
| 文脈の扱い | いま開いている作業と密接 | タスクごとに独立しやすい |
| 運用思想 | 開発者の手元で増幅する | 作業をジョブ化して流す |

### 1-3. 設計思想の違い

#### Claude Code の思想

- ターミナル中心
- ローカルワークフローとの密結合
- Unix 的に composable
- 調査、編集、実行、検証を短いループで回す
- 人間が途中で介入しながら精度を上げる

#### Codex の思想

- エージェントへの委譲を前提
- 並列タスク実行
- バックグラウンド作業
- 検証可能なログや成果物の回収
- 実装だけでなく、自動化やワークフロー化まで視野に入れる

## 2. それぞれの得意領域と不得意領域

### 2-1. Claude Code が得意なこと

- 曖昧な要求を具体化する
- コードベースを読みながら方針を詰める
- ローカル環境依存の調査や再現確認
- 実装の中核を作る
- 細かい修正、統合、仕上げ
- レビュー結果を解釈して現実的に直す

### 2-2. Claude Code が相対的に弱いこと

- 大量の非同期並列タスクを主役として回すこと
- 長時間バックグラウンドで独立作業させること
- 同種の定型作業を多数まとめて処理すること

### 2-3. Codex が得意なこと

- よく定義されたタスクをまとめて任せる
- 複数案を並列に試す
- レビュー、回帰確認、テスト補強
- 大きめの単純作業や反復作業
- 定常業務や自動化
- issue、PR、ドキュメントなどを横断した非同期作業

### 2-4. Codex が相対的に弱いこと

- その場で細かく対話しながら設計を練ること
- ローカル固有事情に強く依存する調査
- 途中で頻繁に方向転換する曖昧タスク
- 人間の感覚的判断を都度求めるような作業

## 3. 私を含めた役割分担

### 3-1. 基本原則

- 私: 意思決定者
- Claude Chat: 構想整理者
- Claude Code: 主実装者
- Codex: 非同期レビュア兼サブ実装者

### 3-2. 私の役割

- 目的を決める
- 優先順位を決める
- 制約を決める
- 受け入れ条件を決める
- 最終採否を決める
