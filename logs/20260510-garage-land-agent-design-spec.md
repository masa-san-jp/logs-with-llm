# ガレージ土地検索エージェント 設計仕様書

**バージョン:** 1.0  
**作成日:** 2026-05-12  
**対象読者:** 開発者・プロダクトオーナー

-----

## 1. 概要

### 1.1 目的

ガレージ投資用土地の選定に必要な情報収集・分析を自動化するWebアプリケーション。  
不動産ポータルサイトから土地情報を取得し、ガレージ台数試算と収益利回りを即時算出する。

### 1.2 背景・課題

|課題              |本システムの解決策                     |
|----------------|------------------------------|
|アットホーム等に公開APIがない|Claude の web_search ツールで自然言語検索|
|台数・利回りの手計算が煩雑   |面積・エリア相場から自動試算                |
|複数サイトの横断確認が手間   |1クエリで複数サイトを横断検索               |

## 3. 機能仕様

### 3.1 検索入力

|項目    |型     |必須|説明                  |
|------|------|--|--------------------|
|エリア   |string|✅ |都道府県・市区町村（例: 埼玉県川越市）|
|予算（最小）|number|- |万円単位                |
|予算（最大）|number|- |万円単位                |
|最低面積  |number|- |㎡単位                 |
|エリア区分 |enum  |✅ |賃料計算に使用（6段階）        |

### 3.2 エリア区分と賃料相場

|エリア区分   |月額賃料想定（円/台）  |
|--------|-------------|
|東京都心    |40,000〜60,000|
|東京郊外    |20,000〜35,000|
|大阪・名古屋中心|25,000〜40,000|
|政令指定都市  |15,000〜25,000|
|地方都市    |8,000〜15,000 |
|郊外・農村   |5,000〜10,000 |


> 賃料相場は月極駐車場・ガレージ賃貸の市場平均値。実際の値は立地・設備・需給により変動する。

### 3.3 試算ロジック

#### ガレージ台数計算

```
有効面積 = 土地面積(㎡) × 0.60
ガレージ台数 = floor(有効面積 / 25)  ※ 最小 1 台
```

- 有効面積率 60%：駐車スペース以外（通路・建築基準法上の後退等）を考慮
- 1台分 25㎡：5m × 5m（標準的な普通車1台分 + 余裕）

#### 表面利回り計算

```
月額平均賃料 = (賃料下限 + 賃料上限) / 2
年間収益 = ガレージ台数 × 月額平均賃料 × 12 × 0.80（稼働率）
表面利回り(%) = 年間収益 / (土地価格[万円] × 10,000) × 100
```

#### 利回り評価基準

|利回り  |表示色|評価  |
|-----|---|----|
|8% 以上|緑  |高利回り|
|5〜8% |黄  |標準  |
|5% 未満|赤  |要検討 |

### 3.4 エリア区分の自動推定

検索結果の `prefecture`・`location` フィールドをもとに、エリア区分を自動判定する。  
ユーザーが手動で設定したエリア区分は**フォールバック**として機能し、自動判定が優先される。

```
判定優先順位:
1. 東京都 + 都心区（千代田・中央・港・新宿・渋谷等）→ 東京都心
2. 東京都（上記以外）→ 東京郊外
3. 大阪市 or 名古屋市 → 大阪・名古屋中心
4. 政令指定都市名に一致 → 政令指定都市
5. "市" を含む → 地方都市
6. その他 → 郊外・農村
```

## 4. API 仕様

### 4.1 Anthropic Messages API

**エンドポイント:** `https://api.anthropic.com/v1/messages`  
**メソッド:** POST

#### リクエスト

```json
{
  "model": "claude-sonnet-4-20250514",
  "max_tokens": 2000,
  "system": "<SYSTEM_PROMPT>",
  "tools": [
    { "type": "web_search_20250305", "name": "web_search" }
  ],
  "messages": [
    { "role": "user", "content": "<検索条件テキスト>" }
  ]
}
```

#### システムプロンプト設計方針

- 役割定義：ガレージ投資専門の不動産エージェント
- 出力形式：**JSON のみ**（前置き・説明文禁止）
- 必須フィールド：`listings[]`・`market_note`
- 件数指定：3〜5件
- 価格・面積不明時：推定値を許容（`notes` に「推定」と記載）

#### レスポンス（JSON スキーマ）

```json
{
  "listings": [
    {
      "title": "string",
      "price": "number | null（万円）",
      "area": "number | null（㎡）",
      "location": "string（市区町村）",
      "prefecture": "string（都道府県）",
      "url": "string | null",
      "source": "string（情報源サイト名）",
      "notes": "string（特記事項）"
    }
  ],
  "market_note": "string（エリア相場コメント）"
}
```

### 4.2 エラーハンドリング

|エラー種別                        |処理               |
|-----------------------------|-----------------|
|API ステータス 4xx / 5xx          |エラーメッセージをログ表示    |
|JSON パース失敗                   |エラーメッセージをログ表示    |
|`listings` フィールド欠損           |専用エラーメッセージ       |
|ユーザーによるキャンセル（AbortController）|「キャンセルしました」ログ表示  |
|マークダウン ````json ```` ブロック混入  |正規表現で除去してから parse|

## 6. 制約・免責

### 6.1 データ精度

- 取得情報はウェブ検索ベースであり、**リアルタイム在庫・正確な価格を保証しない**
- 価格・面積が不明な場合、Claude が推定値を返す場合がある
- 利回り試算は以下を**含まない**：固定資産税、初期工事費、管理費、空室リスク増減

### 6.2 法的・倫理的制約

- 本ツールはスクレイピングではなく、一般公開されたウェブ検索結果を利用する
- 投資判断は専門家・現地確認のうえ行うこと
- 賃料相場データは定期的な見直しが必要

## 8. 用語集

|用語   |定義                                   |
|-----|-------------------------------------|
|表面利回り|年間収益 ÷ 物件取得価格 × 100（%）。諸経費非考慮        |
|有効面積率|土地面積のうちガレージとして活用できる割合（本システムでは 60% 固定）|
|稼働率  |ガレージ契約済み率の想定値（本システムでは 80% 固定）        |
|エリア区分|賃料相場算出のためのエリア分類（6段階）                 |
