# パイプライン構築に必要な情報

## 1. 業務インベントリ（最優先）

- 現在行っているすべての業務タスクのリスト
- 各タスクの週次・月次の所要時間
- タスクの実行頻度（毎日/週次/月次/不定期）
- タスクの担当者と関与人数

## 2. タスクの性質分析

- インプット/アウトプットの形式（メール・スプレッドシート・PDF・システム操作など）
- ルールベースで定義できるか、判断が必要か
- 例外ケースの頻度と複雑さ
- 他タスクとの依存関係

## 3. 現在の技術環境

- 使用中のツール・SaaS・社内システム
- APIやデータエクスポート機能の有無
- 自動化ツールの導入可否（RPA・Zapier・Make・Python等）
- IT部門の制約やセキュリティポリシー

## 4. 制約条件

- 自動化に使える予算（ツール費用・開発工数）
- 自動化を実施できる人材スキル（エンジニアがいるか）
- コンプライアンス上、人が介在しなければならないプロセス
- 自動化完了までの期限

## 5. 優先順位の判断基準

- 削減したい時間のターゲット（5時間/週 = どのタスクで達成するか）
- リスク許容度（誤動作した場合の影響度）
- 費用対効果の閾値
