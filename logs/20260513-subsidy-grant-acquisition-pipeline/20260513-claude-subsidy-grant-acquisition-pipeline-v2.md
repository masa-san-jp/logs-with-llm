# 補助金・助成金獲得パイプライン v2

## 全体フロー概観

```
[Phase 0: 自組織・事業マスタ整備]
            ↓（全フェーズの共通インプット）
[1. 情報収集・調査] → [2. 適格性評価] → [3. 申請準備] → [4. 申請・審査対応] → [5. 採択後管理] → [6. 完了・次サイクル]
```

> **Phase 0 は「一度整備して継続更新する台帳」**。各Phaseはここから情報を参照・流用する。

---

## Phase 0｜自組織・事業マスタ整備（基盤レイヤー）

補助金申請の全フェーズで繰り返し参照・転記される情報を一元管理する。
申請のたびにゼロから集める非効率を排除し、申請品質と速度を両立させる目的。

### 0-1. 法人基本情報
| 項目 | 内容例 | 更新頻度 |
|---|---|---|
| 法人名（正式・英語） | 株式会社〇〇 / XX Co., Ltd. | 変更時 |
| 法人番号 | 13桁の番号 | 固定 |
| 設立年月日 | 業歴計算に使用 | 固定 |
| 資本金 | 中小企業判定に使用 | 変更時 |
| 本社所在地 | 都道府県・市区町村 | 変更時 |
| 代表者名・役職 | 申請書・宣誓書に記載 | 変更時 |
| 主たる業種・JSICコード | 補助金の対象業種判定に使用 | 変更時 |
| 許認可・資格（保有一覧） | 建設業許可・ISO認証 など | 変更時 |

### 0-2. 規模・財務サマリ
| 項目 | 内容 | 更新頻度 |
|---|---|---|
| 従業員数（正規・非正規別） | 中小企業・小規模事業者の判定基準 | 年次 |
| 直近3期の売上高 | 成長性・事業規模の説明に使用 | 年次 |
| 直近3期の経常利益 | 財務健全性の説明に使用 | 年次 |
| 直近3期の付加価値額 | 賃上げ加点要件の計算に使用 | 年次 |
| 賃上げ計画・実績 | 多くの補助金で加点対象 | 年次 |
| 納税状況（滞納なし確認） | 申請資格要件 | 年次 |
| 直近の設備投資履歴 | 重複投資・補助対象外の確認用 | 都度 |

### 0-3. 事業・強み・課題の構造化
| 項目 | 内容 |
|---|---|
| 主力事業の概要（200字程度） | 申請書の「事業概要」欄に流用 |
| ターゲット市場・顧客 | 市場環境記述に使用 |
| 競合優位性・独自技術 | 事業の差別化・革新性の記述に使用 |
| 現在の経営課題トップ3 | 補助金の「解決したい課題」との適合確認に使用 |
| 中期経営計画（方向性） | 補助事業との整合性説明に使用 |
| 過去の補助金受給履歴 | 重複・制限確認、実績アピールに使用 |

### 0-4. 定型書類の最新版一元管理
| 書類 | 保管場所 | 有効期限管理 |
|---|---|---|
| 登記簿謄本 | 共有フォルダ | 3ヶ月以内のもの（申請都度取得） |
| 印鑑証明 | 共有フォルダ | 3ヶ月以内のもの |
| 直近2〜3期の決算書（税務署受付印付） | 共有フォルダ | 年次更新 |
| 納税証明書（その1・その2） | 共有フォルダ | 年次更新 |
| 会社概要・パンフレット | 共有フォルダ | 随時更新 |

### 0-5. 申請書・事業計画書テンプレート
- 過去に採択された申請書の匿名化・汎用化テンプレート
- 不採択時の審査講評と改善コメントの蓄積
- 補助金カテゴリ別（設備投資系・IT導入系・研究開発系）のひな形

## Phase 1｜情報収集・調査

### 1-1. ソース登録・モニタリング
- 公的機関（経産省・中小機構・都道府県・市区町村）の公募情報を定期チェック
- J-Net21、ミラサポplus、e-Gov などのポータルをRSS・メールで購読
- 業界団体・商工会議所からの案内を受信

### 1-2. 公募情報の一次整理
| 収集項目 | 内容 |
|---|---|
| 補助金名 | 正式名称・通称 |
| 主管機関 | 省庁・自治体・財団 |
| 対象者要件 | 業種・規模・設立年数 など |
| 対象経費 | 設備・人件費・委託費 など |
| 補助率・上限額 | 例：2/3補助・上限1,000万円 |
| 公募期間 | 締切日・受付回数 |
| 採択率（過去実績） | あれば取得 |

---

## Phase 2｜適格性評価（スクリーニング）

> **Phase 0 の情報を参照して機械的・迅速に判定する**

### 2-1. 申請資格の確認
- 0-1（業種コード・法人格）→ 対象者要件と照合
- 0-2（従業員数・資本金）→ 中小企業・小規模事業者の判定
- 0-2（納税状況・補助金受給履歴）→ 申請制限・重複受給の確認

### 2-2. 事業適合度の評価
- 0-3（経営課題・中期計画）→ 補助対象事業との一致度スコアリング
- 0-3（競合優位性・独自技術）→ 審査項目「革新性・優位性」との対応付け
- 費用対効果試算（申請工数 vs 獲得見込み額）

### 2-3. 優先順位付け
- スコア上位案件を申請候補リスト化
- 締切・準備期間を考慮したタイムライン確認

---

## Phase 3｜申請準備

> **Phase 0 の情報・書類・テンプレートを流用して準備工数を最小化する**

### 3-1. 事業計画の策定・ブラッシュアップ
- 0-3（課題・強み）をベースに「現状課題→補助事業→期待効果」の論理構成
- 0-2（財務データ）を使ったKPI・数値目標の設定
- 0-5（過去申請テンプレート）をベースに申請書を作成

### 3-2. 必要書類の収集・作成
- 0-4（定型書類）から有効期限内のものを確認・流用
- 有効期限切れのものを追加取得

| カテゴリ | 代表的な書類 |
|---|---|
| 法人基本 | 登記簿謄本・定款・印鑑証明 |
| 財務 | 直近2〜3期の決算書・納税証明 |
| 事業計画 | 申請書本体・事業計画書・実施スケジュール |
| 見積 | 対象経費の相見積書（2社以上が多い） |
| その他 | 宣誓書・同意書・特定書類（補助金ごとに異なる） |

### 3-3. 予算・資金計画の整備
- 補助対象経費と対象外経費の分離
- 補助金受給前の立替払い計画（多くは後払い）
- 金融機関との事前相談

### 3-4. 申請書レビュー・品質確認
- 審査基準に対する自己採点
- 第三者（社内・外部専門家）によるレビュー
- 提出前チェックリストの消込

---

## Phase 4｜申請・審査対応

### 4-1. 申請提出
- 電子申請（jGrants 等）または郵送・窓口提出
- 提出完了の証跡保存（受付番号・受領確認メール）

### 4-2. 審査期間中の対応
- 補足資料・追加説明の要求に迅速対応
- 審査員ヒアリング・プレゼンへの備え

### 4-3. 採否通知の確認
- 採択通知：交付申請手続きへ移行
- 不採択通知：審査講評を取得 → **Phase 0-5（テンプレート）に改善内容を反映**

---

## Phase 5｜採択後管理（交付〜実績報告）

### 5-1. 交付申請・交付決定
- **交付決定前の発注・契約・支払いは原則補助対象外**（最重要）
- 交付決定通知の受領と内容確認

### 5-2. 事業実施・経費管理
- 補助事業計画に沿った実施・進捗管理
- 経費の証憑管理（請求書・領収書・振込記録の一式保管）
- 変更が生じた場合は事前に変更承認申請

### 5-3. 実績報告・精算
- 完了報告書・実績報告書の提出
- 経費精算・補助金額確定 → 入金

### 5-4. 事後モニタリング対応
- 補助事業完了後の継続報告（多くは3〜5年間）
- 設備の転用・処分制限の遵守

---

## Phase 6｜完了・ナレッジ蓄積・次サイクル

### 6-1. Phase 0 マスタの更新
- 財務サマリ・従業員数・賃上げ実績を年次更新
- 受給履歴・保有許認可を更新
- 採択書類を 0-5 テンプレートとして格納

### 6-2. 振り返り・改善
- 採択・不採択要因の分析
- 申請工数・コストの記録

### 6-3. 次サイクルへの接続
- 翌年度公募の先行調査
- 中期経営計画の更新と補助金との適合性再確認

## Phase 0 の整備ロードマップ（初回構築目安）

| ステップ | 作業内容 | 目安工数 |
|---|---|---|
| Step 1 | 0-1〜0-2：法人基本・財務情報の入力 | 0.5日 |
| Step 2 | 0-3：事業・強み・課題の構造化（経営者インタビュー） | 1〜2日 |
| Step 3 | 0-4：定型書類の収集・整理・保管場所設定 | 0.5日 |
| Step 4 | 0-5：過去申請書の収集・テンプレート化 | 1日 |
| **合計** | | **3〜4日** |

> 初回整備後は**年1回（決算後）の定期更新**と**申請イベントごとの部分更新**で運用する。

## 主要な落とし穴（リスク一覧）

| リスク | 内容 | 対策 |
|---|---|---|
| 交付決定前発注 | 補助対象外となり全額自己負担 | 交付決定日を必ず確認してから発注 |
| 証憑不備 | 精算時に経費が認められない | 発生都度、証憑セットで保管 |
| 報告義務漏れ | 返還請求・指定取消のリスク | 報告スケジュールをカレンダー管理 |
| 重複受給 | 同一経費への複数補助金の適用不可 | Phase 0-2 の受給履歴で都度確認 |
| 申請書の抽象的記述 | 審査で低評価 | Phase 0-3 の数値・根拠を必ず引用 |
| 基礎情報の陳腐化 | 古い財務データで適格性を誤判定 | Phase 0 の年次更新ルールを遵守 |
