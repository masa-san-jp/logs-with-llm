# 補助金・助成金獲得パイプライン｜戦略計画・ポートフォリオ管理レイヤー

## 全体フロー（追加後）

```
[Phase S: 年次戦略計画]
        ↓ 目標・KPI・ポートフォリオ方針の設定
[Phase 0: 自組織・事業マスタ整備]
        ↓ 全フェーズの共通インプット
[Phase P: ポートフォリオ管理]  ←→  [1.調査] → [2.評価] → [3.準備] → [4.申請] → [5.採択後] → [6.完了]
        ↑___________________________フィードバック・実績反映_________________________________↑
```

> Phase S は年1回（期初）に実施。Phase P は通年で継続運用し、各申請の進捗・確度・金額を一元管理する。

---

## Phase S｜年次戦略計画

### S-1. 目標金額の設定

#### 目標金額の決め方（トップダウン）
1. **投資計画から逆算する**
   - 当期・来期に予定している設備投資・IT投資・人材投資の総額を確認
   - そのうち補助金で賄いたい割合を設定（例：設備投資2,000万円のうち50% = 1,000万円）

2. **資金調達計画の中での位置づけを決める**
   - 自己資金・融資・補助金・助成金の調達ミックスを明確化
   - 補助金は「後払い」が原則のため、一時立替資金の手当てと分けて計画する

3. **目標金額の分解**

| 区分 | 内容 | 目標金額（例） |
|---|---|---|
| 設備・システム投資補助 | ものづくり補助金・IT導入補助金など | ¥8,000,000 |
| 雇用・人材系助成金 | キャリアアップ助成金・人材開発支援など | ¥3,000,000 |
| 研究開発・実証 | SBIR・地域イノベーションなど | ¥5,000,000 |
| 地方自治体単独 | 都道府県・市区町村の独自補助 | ¥2,000,000 |
| **合計** | | **¥18,000,000** |

#### 目標設定の現実化チェック
- 過去の採択実績・採択率から「申請すべき件数」を逆算する
- 例：目標1,000万円、平均採択額250万円、採択率40% → 必要申請数 = 1,000万 ÷ (250万 × 40%) = **10件**
- 自組織の申請工数キャパシティと照合し、必要なら外部専門家の活用を計画に含める

### S-2. ポートフォリオ方針の設定

目標金額を1本の大型補助金に集中させるリスクを回避し、複数の補助金に分散する方針を決める。

#### 分散軸の設計

| 分散軸 | 内容 |
|---|---|
| **規模の分散** | 大型（500万円超）・中型（100〜500万円）・小型（〜100万円）を組み合わせる |
| **主管機関の分散** | 国・都道府県・市区町村・財団で分散し、政策変更リスクを軽減 |
| **締切時期の分散** | 年間を通じて申請が平準化されるよう時期をずらす |
| **採択難易度の分散** | 高難度（確度20〜40%）・中難度（40〜60%）・低難度（60%超）を混在させる |
| **補助対象経費の分散** | 設備・IT・人件費・委託費など、対象経費カテゴリを分散させる |

#### リスクバッファの設定
- 目標金額に対して **1.5〜2倍の申請金額** をパイプラインに積む
- 例：目標1,000万円 → パイプライン積み上げ目標 1,500〜2,000万円分

### S-3. 年次カレンダーの作成

主要補助金の公募スケジュールを年間カレンダーに落とし込む。

| 月 | 公募予定（例） | 申請準備開始目安 |
|---|---|---|
| 4月 | ものづくり補助金（第〇次） | 2月から |
| 5月 | IT導入補助金（通年） | 随時 |
| 6月 | 事業再構築補助金（第〇次） | 4月から |
| 9月 | 小規模事業者持続化補助金 | 7月から |
| 11月 | 各省庁来年度公募先行情報収集 | ― |
| 1〜3月 | 自治体単独補助（年度末公募が多い） | 12月から |

> 実際の公募スケジュールは年度ごとに変動するため、Phase 1（情報収集）の結果で随時更新する。

### S-4. 年次KPIの設定

| KPI | 計算式 | 目標例 |
|---|---|---|
| 申請件数 | 年間の申請提出数 | 10件 |
| 採択件数 | 採択された件数 | 4件 |
| 採択率 | 採択件数 ÷ 申請件数 | 40% |
| 申請総額 | 申請した補助金の合計額 | 2,000万円 |
| 採択総額 | 採択された補助金の合計額 | **1,000万円（目標）** |
| 入金額 | 実際に受け取った補助金額 | 800万円（タイムラグあり） |
| 申請ROI | 採択総額 ÷ 申請工数コスト | 10倍以上 |

## Phase P｜ポートフォリオ管理（通年運用）

各申請案件を「進捗ステージ × 確度 × 期待値」で一元管理し、目標金額の達成見通しをリアルタイムで把握する。

### P-1. 案件管理台帳の設計

各補助金案件に以下の属性を付与して管理する。

| 項目 | 内容 |
|---|---|
| 補助金名 | 正式名称 |
| ステージ | 下記ステージ定義を参照 |
| 申請額（見込み） | 申請予定の補助金額 |
| 採択確度（%） | 担当者の主観評価 or 過去採択率をベース |
| 期待値 | 申請額 × 採択確度 |
| 締切日 | 公募締切 |
| 担当者 | 社内担当・外部専門家 |
| 入金予定時期 | 採択後の精算・入金見込み月 |
| 備考 | 加点要件・リスク・依存関係 |

#### ステージ定義

```
S0: 候補リスト（情報収集済み・評価前）
S1: 適格性評価中
S2: 申請準備中
S3: 申請済み・審査中
S4: 採択・交付申請中
S5: 事業実施中
S6: 実績報告済み・入金待ち
S7: 入金完了（クローズ）
S8: 不採択（クローズ）
```

### P-2. 目標達成見通しの管理

#### パイプライン期待値の計算

| 補助金名 | ステージ | 申請額 | 採択確度 | 期待値 | 入金予定 |
|---|---|---|---|---|---|
| ものづくり補助金 | S3 | 500万円 | 40% | 200万円 | 来期Q1 |
| IT導入補助金 | S2 | 150万円 | 65% | 98万円 | 当期Q4 |
| キャリアアップ助成金 | S5 | 80万円 | 80% | 64万円 | 当期Q3 |
| 都補助金A | S1 | 200万円 | 50% | 100万円 | 来期Q2 |
| …（以下続く） | | | | | |
| **合計** | | **1,800万円** | ― | **850万円** | |

- **パイプライン総額（申請額合計）** が目標の1.5〜2倍になっているかを常時確認
- **期待値合計** が目標金額を下回ったら、S0〜S1の案件を積み増す

#### 信号管理（RAGステータス）
| 状態 | 条件 | アクション |
|---|---|---|
| 🟢 Green | 期待値合計 ≥ 目標金額 | 現状維持・品質向上に注力 |
| 🟡 Yellow | 期待値合計が目標の70〜99% | 新規案件の発掘・追加申請を検討 |
| 🔴 Red | 期待値合計が目標の70%未満 | 大型案件の追加・外部専門家の活用・目標の見直し |

### P-3. 入金キャッシュフロー計画

補助金は後払いが原則のため、**申請額・入金タイミング・立替資金**を一体で管理する。

| 月 | 立替支出予定 | 入金予定（補助金） | 差引資金需要 |
|---|---|---|---|
| 4月 | ¥5,000,000 | ¥0 | ▲¥5,000,000 |
| 7月 | ¥2,000,000 | ¥800,000 | ▲¥1,200,000 |
| 10月 | ¥1,000,000 | ¥3,000,000 | +¥2,000,000 |
| 1月 | ¥500,000 | ¥2,000,000 | +¥1,500,000 |

- 立替資金が大きい月は金融機関との事前折衝（つなぎ融資など）を計画に組み込む

### P-4. ポートフォリオの定期レビュー

| レビュー種別 | 頻度 | 確認内容 |
|---|---|---|
| 週次チェック | 毎週 | 期限・提出物・ステージ変化 |
| 月次レビュー | 毎月 | 期待値合計・RAGステータス・新規案件の追加 |
| 四半期レビュー | 年4回 | KPI進捗・ポートフォリオ方針の見直し |
| 年次レビュー | 年1回 | KPI実績評価・翌年度 Phase S の策定 |

## 改訂後の全体パイプライン構造まとめ

```
Phase S  年次戦略計画          ← 年1回：目標金額・KPI・ポートフォリオ方針の設定
   ↓
Phase 0  自組織マスタ整備       ← 年次更新：法人基本・財務・事業・書類・テンプレート
   ↓
Phase P  ポートフォリオ管理     ← 通年運用：案件台帳・期待値管理・CF計画・定期レビュー
   ↕（双方向）
Phase 1  情報収集・調査
Phase 2  適格性評価
Phase 3  申請準備
Phase 4  申請・審査対応
Phase 5  採択後管理
Phase 6  完了・ナレッジ蓄積     → Phase S（翌年度計画）へフィードバック
```

## 目標金額から申請件数を逆算するシミュレーション

| 想定採択率 | 平均採択額 | 目標獲得額 | 必要申請件数 |
|---|---|---|---|
| 30% | 200万円 | 1,000万円 | 17件 |
| 40% | 250万円 | 1,000万円 | 10件 |
| 50% | 300万円 | 1,000万円 | 7件 |
| 40% | 250万円 | 2,000万円 | 20件 |

> 採択率・平均採択額は Phase 0-5（過去申請実績）と業界平均（各補助金の公表採択率）から設定する。
> 必要申請件数が自組織のキャパを超える場合は、**外部専門家（中小企業診断士・行政書士）への委託コスト**を目標金額から逆算して予算化する。
