# hermes-agent のヘッドレス呼び出しとスキル化（`hermes -z "prompt"`）

## 1. `hermes -z "prompt"` とは（ヘッドレス / One-shotモード）

`hermes -z` は **完全に非対話型の1回限り実行コマンド** です。
インタラクティブCLI（`hermes` だけ実行）とは異なり、余計な出力（バナー、スピナー、途中経過など）を一切出さず、**最終回答のみをプレーンテキストでstdout** に出力します。

### 主な特徴

- シェルスクリプト、cron、他のAIツールからの呼び出しに最適
- 標準入力からもプロンプトを受け取れる
- X検索ツール（x_search）が有効化されていれば、X Premiumプランだけでリアルタイム検索が可能

### 使用例

```bash
# 基本的な実行
hermes -z "現在の日付と時間を教えてください"

# X検索を実行する例
hermes -z "x_searchツールを使って『AI エージェント』に関する直近24時間の投稿を5件検索し、要約せよ"

# パイプ入力
hermes -z "このコードをレビューして改善点をリストアップしてください" < main.py
```

### 便利なオプション

| オプション | 説明 |
|---|---|
| `--toolsets x_search` | X検索ツールのみ有効化 |
| `--model grok-4.3` | 使用モデル指定 |
| `--quiet` | さらに出力を最小化 |
| `-s skill-name` | 特定のスキルを事前ロード |

## 2. 「スキル化」とは

hermes-agent の強力な機能の一つで、再利用可能な手順書（スキル）を作成・登録できます。

### スキル作成手順

1. `~/.hermes/skills/` フォルダ内にスキル名フォルダを作成
2. `SKILL.md`（または `skill.yaml`）を配置
3. 自然言語で指示・ツール使用ルール・出力形式を記述

### X検索専用スキルの例（`SKILL.md`）

```markdown
# Skill: x-search

## Description
X（Twitter）で高度検索を行い、要約を返す専用スキル

## Usage
/x-search <クエリ> [since:YYYY-MM-DD]

## Instructions
- 常に x_search ツールを使用
- advanced operators（from:、since:、filter:media など）を適切に活用
- 結果は以下の形式で出力：
  - 要約（3〜5行）
  - 主要投稿の抜粋（最大5件）
  - 各投稿に引用URLを付与
- 最新情報に偏りすぎないようバランスを取る
```

スキル登録後、`hermes -z "x-search hermes-agent 最新動向"` のように簡潔に呼び出せます。

## 3. Codex / Claude Code などからX検索を活用する方法

CodexやClaude Codeなどの外部AIコーディングエージェントから、`hermes -z` を外部ツール/スキルとして統合できます。

### 統合パターン

1. **シェルコマンドとして直接呼び出し**
   - Claude CodeやCodexのツール定義で `hermes -z "...${query}..."` を実行

2. **専用ラッパースクリプト作成**

    ```bash
    #!/bin/bash
    # x-search-wrapper.sh
    hermes -z "x_searchツールを使って '${1}' を検索し、要約せよ"
    ```

3. **メタエージェントとして利用**
   - 「最新のX情報を調べてからコードを書け」→ hermesに委譲

### メリット

- Codex/Claude単体では弱いリアルタイムX検索を低コストで補完
- スキル化により毎回同じプロンプトを書く必要がなくなる
- 出力形式が安定（deterministic）になる

## まとめと推奨

### まずは動作確認

```bash
hermes -z "x_searchツールを使ってAIエージェントに関する最近の投稿を要約せよ"
```

### スキル作成のコツ

出力形式を厳密に指定すると再現性が高まる。

### トラブルシューティング

| 症状 | 対処 |
|---|---|
| OAuth切れ | `hermes tools` で再認証 |
| リモート環境での接続失敗 | SSHトンネル（ポート56121）を使用 |

---

この方法により、X Premiumプランだけで強力なX検索を各種AIワークフローに簡単に組み込めます。
