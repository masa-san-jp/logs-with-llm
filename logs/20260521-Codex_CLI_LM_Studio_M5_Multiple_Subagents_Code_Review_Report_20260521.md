# Codex CLI と LM Studio を活用した  
MacBook Pro M5 上での複数サブエージェント並列実行による  
コードレビューおよびバグ発見の包括的レポート

**作成日**: 2026年5月21日  
**対象読者**: Apple Silicon（特に MacBook Pro M5）搭載の macOS 環境で、ローカル環境にてプライバシーを完全に保護しながら高性能な人工知能エージェントを運用したい開発者の方々  
**目的**: 本レポートは、X（旧 Twitter）上の投稿（投稿ID: 2057186643086278782）で紹介されたデモンストレーションを基に、Codex CLI、LM Studio、および MLX エンジンのベータ版バッチング機能を組み合わせた複数サブエージェントの並列実行手法を、専門用語を一切省略せずに徹底的に丁寧に解説するものです。不足している情報は公式ドキュメントおよび最新の技術情報に基づいて補足し、実際に再現可能な手順書レベルでまとめています。

## 1. 本セットアップの概要と背景

本手法は、**OpenAI が提供する Codex CLI**（端末上で動作するコーディングエージェント）と**LM Studio**（ローカルで大規模言語モデルを管理・実行するためのグラフィカルユーザーインターフェース付きアプリケーション）を組み合わせ、**Apple Silicon 向けの機械学習フレームワークである MLX を基盤とした MLX エンジン**のベータ版で追加された**連続バッチ処理機能**（continuous batching）を活用します。

これにより、**1つの Qwen 3.6 モデル**（Alibaba Cloud が開発した Qwen シリーズの最新版大規模言語モデル）を共有しながら、**複数のサブエージェント**（メインエージェントが明示的に指示して生成する、専門化された独立したエージェント）を同時に実行し、コードベース全体のレビュー、バグ発見、セキュリティ脆弱性検出を高速かつ並列に行うことが可能になります。

**最大の利点**:
- クラウドサービスへの依存がなく、**完全にローカルで動作**するため、機密性の高い企業コードや個人プロジェクトの知的財産を外部に送信せずに済みます。
- 連続バッチ処理機能により、従来の逐次処理では避けられなかった待ち時間が大幅に短縮され、MacBook Pro M5 の統一メモリ（Unified Memory）を効率的に活用できます。
- コストはゼロ（電力消費のみ）で、プライバシーが完全に保護されます。

このデモンストレーションは、2026年5月20日に投稿された動画で確認された通り、MacBook Pro M5 上で Qwen 3.6 モデルを用いてサブエージェントがリアルタイムにファイル探索・分析・結果集約を行う様子を示しています。

## 2. 必要な前提環境（詳細仕様）

- **ハードウェア**: MacBook Pro M5（または M4 / M3 シリーズ以上）。統一メモリは最低 32GB を強く推奨し、64GB 以上が快適です。Qwen 3.6 の 30B クラスモデルを 4bit 量子化で実行する場合、約 20〜30GB のメモリを消費します。
- **オペレーティングシステム**: macOS の最新安定版（Neural Engine および MLX の最適化を最大限活用するため、macOS 26.2 以降を推奨）。
- **ソフトウェア**:
  - LM Studio（最新版 0.4.2 以降。Developer Mode を有効化し、Runtime Channel を Beta に設定）。
  - Codex CLI（OpenAI 公式の `@openai/codex` パッケージ。npm 経由でグローバルインストール）。
  - Qwen 3.6 モデル（LM Studio 内で MLX 形式の 4bit / 6bit 量子化版をダウンロード・ロード）。

## 3. 詳細なセットアップ手順

### 3.1 LM Studio のインストールと MLX エンジンのベータ版有効化

1. 公式サイト（https://lmstudio.ai）より LM Studio をダウンロード・インストールします。
2. アプリケーションを起動後、**設定（Settings）** → **Developer** セクションで **Developer Mode** をオンにします。
3. **Runtime Channel** を **Beta** に切り替えます。
4. コマンドラインまたは UI から MLX エンジンを更新します：
   ```
   lms runtime update mlx
   ```
   これにより、**連続バッチ処理機能**（複数の推論リクエストを1回のフォワードパスで同時に処理する仕組み）が有効化され、キー・バリューキャッシュ（KV Cache）の切り替えオーバーヘッドが最小化されます。
5. **Qwen 3.6 モデル** を検索・ダウンロードします（例: `qwen/qwen3.6-30b` または `qwen/qwen3.6-35b` の MLX 最適化版）。
6. モデルロード時に **Advanced Settings** で **Max Concurrent Predictions**（同時予測最大数）を 2〜4 以上に設定します（メモリに余裕がある場合）。
7. サーバーを起動します：
   ```
   lms server start --port 1234
   ```
   （UI の「Start Server」ボタンでも可能です。これで OpenAI 互換の `/v1` エンドポイントが http://localhost:1234/v1 で公開されます。）

### 3.2 Codex CLI のインストールと LM Studio への接続設定

1. 端末で以下のコマンドを実行して Codex CLI をグローバルインストールします：
   ```
   npm install -g @openai/codex
   ```
2. ホームディレクトリに設定ファイル `~/.codex/config.toml` を作成・編集します。内容は以下の通りです：
   ```toml
   [model_providers.lm_studio]
   name = "LM Studio"
   base_url = "http://localhost:1234/v1"

   [profiles.qwen3-6-mlx-high]
   model_provider = "lm_studio"
   model = "qwen/qwen3.6-30b-mlx-high"  # 実際にロードしたモデル名に合わせて調整
   temperature = 0.7
   max_tokens = 8192
   ```
3. プロジェクトディレクトリで Codex CLI を起動します：
   ```
   codex --profile qwen3-6-mlx-high --cwd /path/to/your/project
   ```

### 3.3 複数サブエージェントの並列実行手順

1. Codex CLI の対話セッション内で、メインエージェントに対して明確に指示します：
   ```
   Review this entire codebase for bugs, security vulnerabilities, performance issues, and maintainability improvements. Spawn 3 sub-agents in parallel: one specialized in backend logic (app.py), one in frontend (index.html), and one in security scanning. Perform thorough parallel analysis and consolidate the results.
   ```
2. Codex CLI は自動的にサブエージェントを生成します。各サブエージェントは独立したコンテキストを持ち、ファイル読み込み・静的解析・ログ分析を同時に行います。
3. サブエージェントの管理コマンド：
   - `/agents` または `/agent`：アクティブなサブエージェント一覧を表示し、切り替えや停止を行います。
4. 動画デモンストレーションと同様のログが表示されます：
   - ファイル探索（Explored）
   - サブエージェント生成（Spawned ...）
   - 並列処理中（Waiting for agents...）
   - 結果集約：Critical / Medium / Low バグ、セキュリティ脆弱性リスト、修正提案

連続バッチ処理機能により、複数のサブエージェントからの同時リクエストが効率的に処理され、全体のスループットが 2〜5 倍向上します。

## 4. 適合するユースケース（具体的に絞り込み）

本セットアップは**読み込み中心・分析中心の反復作業**に特に適しています。以下に優先度順で具体的に絞り込みます：

1. **コードレビューおよびバグ・セキュリティ発見**（最も適合）  
   プルリクエスト差分や大規模リポジトリ全体を複数視点（セキュリティ専門・バグ専門・パフォーマンス専門）で並列レビュー。

2. **大規模リファクタリング準備**  
   数百ファイル規模のライブラリ更新や API 移行時に、探索担当・修正提案担当・テスト更新担当のサブエージェントを同時稼働。

3. **ユニットテスト生成およびカバレッジ向上**  
   各モジュールごとにサブエージェントを割り当て、テストコードを並列生成。

4. **複雑バグの調査・デバッグ**  
   フロントエンド／バックエンド連携バグをトレース・再現シナリオ作成・ログ分析で並列処理。

5. **新規リポジトリのオンボーディング**  
   大規模コードベースの構造把握とドキュメント自動生成。

**特に向いている開発者・プロジェクト**:
- 個人開発者・インディー開発者（コストゼロ・プライバシー重視）
- 中〜大規模コードベースを扱う Mac ユーザー
- 毎日繰り返されるレビュー作業の自動化を望む方

**避けるべきケース**:
- 完全に新規の創造的機能実装（生成より分析が得意）
- インターネット接続を必須とする外部ツール連携

## 5. パフォーマンス最適化と注意点

- **メモリ管理**: モデルサイズに応じて量子化レベルを調整（4bit が推奨）。M5 の Neural Engine を最大限活用。
- **トラブルシューティング**:
  - サーバー接続エラー → LM Studio サーバーの再起動。
  - 速度低下 → Max Concurrent Predictions を調整、またはモデルを軽量化。
- **セキュリティ**: 完全にローカル実行のため、データ漏洩の心配はありません。
- **拡張性**: カスタムエージェント定義ファイル（`~/.codex/agents/`）を作成することで、役割をさらに細かく特化可能。

## 6. 結論と推奨

本セットアップは、**ローカル環境で実用レベルの複数 AI エージェントチームを運用**する画期的な手法です。MacBook Pro M5 の性能と MLX エンジンの連続バッチ処理機能が融合することで、従来クラウド依存だった高度なコード分析が手元の端末だけで実現可能になりました。
