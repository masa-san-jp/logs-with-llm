# Claude Design 実務導入ガイド

## 前提条件

- Claude Pro以上のプラン（$20/月〜）が必要
- **ブラウザのみ対応**。デスクトップアプリ非対応
- Enterpriseプランはデフォルト無効のため管理者による有効化が必要

-----

## アクセス方法

ブラウザで `https://claude.ai/design` に直接アクセスする。

画面は左ペイン（チャット）と右ペイン（キャンバス）の2分割構成。

-----

## 初回のみ：デザインシステムのセットアップ（約1〜2時間）

### 目的

このステップをスキップすると自社ブランドと無関係な汎用デザインが出力される。一度設定すれば以降のすべてのプロジェクトに自動適用される。

-----

### ステップ0：アセットの準備

Claude Designに渡す前に、以下のアセットを手元に揃える。

**デザイナーや外注経験がない場合、これらが何かわからないことが多い。以下に状況別の準備方法を示す。**

-----

#### ケースA：既存のWebサービス・アプリがある場合（最短・推奨）

既存のプロダクトが一番のアセットになる。以下を用意する。

**1. スクリーンショット（最低5〜10枚）**

以下の画面を優先してキャプチャする。

|優先度|画面           |理由                 |
|---|-------------|-------------------|
|高  |トップ/ランディングページ|ブランドの第一印象が最も反映されている|
|高  |ダッシュボード/メイン画面|UIコンポーネントが最も多く含まれる |
|高  |ボタン・フォームが多い画面|インタラクションの基準になる     |
|中  |エラー画面・空状態    |カラーパレットの全域が確認できる   |
|中  |モバイル表示       |レスポンシブの基準になる       |

Macなら `Cmd + Shift + 4`、Windowsなら `Win + Shift + S` でキャプチャできる。

**2. コードベース（GitHubリポジトリがある場合）**

リポジトリ全体ではなく、以下のいずれかのディレクトリパスだけを指定する。

```
# Tailwind を使っている場合
tailwind.config.js または tailwind.config.ts

# コンポーネントライブラリがある場合
/src/components/
/packages/design-system/
/apps/marketing/

# CSS変数を定義しているファイル
/src/styles/globals.css
/src/styles/variables.css
```

モノレポ全体を渡すとブラウザが重くなるため、必ず上記のようなサブディレクトリを指定する。

-----

#### ケースB：既存プロダクトがなく、新規で始める場合

この場合はアセットを**自分で作る**必要がある。以下の3点だけ用意すれば最低限動く。

**1. カラーコード（最低2色）**

使いたい色のHEXコード（`#1A2B3C` のような6桁の英数字）を用意する。

- 決まっていない場合：競合他社やベンチマークにしたいサービスのサイトをブラウザで開き、デベロッパーツール（F12）で `color` や `background-color` の値をコピーする
- もしくはClaude Designの初回プロンプトで「ブランドカラーはまだ決まっていない。〇〇系の配色で提案して」と伝えて自動生成させる

最低限必要なのは以下の2色。

|役割       |説明                 |例            |
|---------|-------------------|-------------|
|プライマリカラー |ブランドの主要色。ボタンやリンクに使う|`#2563EB`（青系）|
|ニュートラルカラー|テキストや背景に使うグレー系     |`#111827`    |

**2. フォント名（最低1つ）**

使いたいフォント名を決める。Google Fontsから無料で選べる。決まっていない場合は以下から選ぶ。

|印象             |フォント名                       |
|---------------|----------------------------|
|モダン・テック系       |Geist, Plus Jakarta Sans    |
|クリーン・ビジネス系     |DM Sans, Outfit             |
|信頼感・エンタープライズ系  |IBM Plex Sans, Source Sans 3|
|親しみやすい・コンシューマー系|Nunito, Quicksand           |

**3. ロゴファイル（あれば）**

PNG または SVG 形式のロゴファイル。なければスキップして後から追加できる。

-----

#### ケースC：既存のブランドガイドライン・スライドがある場合

過去に外注デザイナーやエージェンシーに依頼した成果物があれば、それをそのまま渡せる。

渡せるもの：

- ブランドガイドラインのPDF
- 既存のピッチデック（PPTX・PDF）
- マーケティング資料・会社紹介スライド

Claude がこれらからカラー・フォント・レイアウトパターンを自動抽出する。

-----

### ステップ1：組織を作成または選択する

1. `https://claude.ai/design` を開く
1. 画面左下の組織名をクリック
1. 既存の組織を選択、または新規作成
1. 表示されるオンボーディングフローを完了する

-----

### ステップ2：アセットをアップロードする

ステップ0で用意したものをアップロードする。複数あるほど精度が上がるが、1点だけでも動作する。

-----

### ステップ3：生成されたデザインシステムを確認する

Claude が以下を自動抽出する。

- カラーパレット（プライマリ・セカンダリ・アクセント）
- タイポグラフィ（フォントファミリー・サイズ・ウェイト）
- コンポーネント（ボタン・カード・ナビゲーション等）
- レイアウトパターン（グリッド・スペーシング）

確認用プロンプト例：

```
「[プロダクト名]のランディングページを作成して」
「[関連指標]を表示するダッシュボードをデザインして」
```

出力が自社ブランドと合致していれば次へ進む。合っていなければ追加アセットをアップロードして再生成する。

-----

### ステップ4：公開する

「Published」トグルをオンにする。これでチームメンバー全員のプロジェクトに自動適用される。

-----

## 通常の開発フロー

```
仕様・PRD・スクリーンショット・口頭のアイデア
                ↓
        新規プロジェクト作成
    （デザインシステムが自動適用される）
                ↓
        コンテキストを追加（任意）
    スクリーンショット・コードベース・競合URL
                ↓
            プロンプト入力
                ↓
        キャンバスで初稿を確認
                ↓
        チャット or インラインコメントで修正
                ↓
        Export → Handoff to Claude Code
                ↓
            Claude Codeが実装
```

-----

## 効果的なプロンプトの書き方

150〜300語が最適。以下の4要素を含める。

```
目的：[何を作るか]
対象ユーザー：[誰が使うか]
必要な要素：[具体的なコンポーネントや情報]
制約：[モバイルファースト・既存コンポーネントを使う 等]
```

**具体例：**

```
月次売上を地域・プロダクトラインでフィルタリングできる
ダッシュボードを作成して。既存のCardコンポーネントと
Primaryボタンを使用。モバイルとデスクトップ両対応。
```

-----

## 修正の使い分け

|操作       |使いどころ         |例                               |
|---------|--------------|--------------------------------|
|チャット     |構造・全体的な変更     |「カラースキームをダークにして」「メトリクスを上段に移動して」 |
|インラインコメント|特定コンポーネントの局所変更|「このボタンのパディングを大きく」「ここをドロップダウンに変更」|

**既知の不具合：** インラインコメントが消えることがある。その場合はチャットに同じ内容を貼り付けて送信する。

-----

## Claude Codeへのハンドオフ

1. 画面右上の「Export」ボタンをクリック
1. 「Handoff to Claude Code」を選択
1. 以下の2択：
- **Send to local coding agent**：コマンドをコピーしてローカルのClaude Codeターミナルにペースト
- **Send to Claude Code Web**：ブラウザ版のClaude Codeに直接送信

バンドルにはデザインファイル・チャット履歴・READMEが含まれており、Claude Codeがリポジトリを既に把握していれば実装精度がさらに上がる。

## やってはいけないこと

- デザインシステム未設定でプロトタイプを作り始める
- 価格・統計・法的文言などの数値テキストを無確認で本番公開する（ハルシネーションのリスクあり）
- Claude Designで細部まで作り込もうとする（探索フェーズで止めてClaude Codeに渡す）
- モノレポ全体をリポジトリ連携する（サブディレクトリを指定する）
- デスクトップアプリで開こうとする（ブラウザのみ対応）
