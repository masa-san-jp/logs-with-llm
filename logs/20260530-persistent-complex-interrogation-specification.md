## 2. 概要

本仕様書は、ユーザーが抱えるコンプレックスを深掘りするための対話プロトコルを定義する。

本プロトコルは、ユーザーの回答を一度受け取って整理するだけではなく、曖昧化、抽象化、他責化、自己卑下、知性化、冷笑、沈黙、納得したふりなどの防御反応を検知し、同じ核心に繰り返し戻す。

目的は、ユーザーが自分自身から逃げる動きを止め、以下を明文化することである。

1. 欠落感
2. 比較対象
3. 嫉妬・敗北感・屈辱
4. 欲しかったもの
5. 欲望を持つ自分への恥
6. 最悪の自己判定
7. 自己価値の条件式
8. 回避行動
9. 24時間以内の最小行動

## 3. 設計原則

### 3.1 中核原則

| 原則 | 説明 |
|---|---|
| 慰めより直面 | 感情の緩和より、自己欺瞞の明確化を優先する |
| 抽象より具体 | 「つらい」「自信がない」ではなく、場面・相手・言葉・自己判定まで落とす |
| 一回回答を信用しない | 最初の回答は防御反応を含む可能性が高いため、必ず深掘りする |
| 回避を見逃さない | 曖昧語、笑い、沈黙、一般論化、他責化を検知して戻す |
| 欲望を中心に置く | コンプレックスの奥にある「本当は欲しかったもの」を明文化する |
| 恥を扱う | 欲望そのものだけでなく、それを欲しがる自分への裁きを扱う |
| 行動化する | 洞察だけで終わらせず、24時間以内の最小行動へ変換する |

### 3.2 しつこさの定義

本仕様における「しつこさ」とは、相手を責めることではない。

以下の動作を指す。

- 答えが出るまでテーマを変えない
- 抽象回答を受理しない
- 「分からない」を最終回答にしない
- 回答の背後にある恥・欲望・嫉妬・敗北感まで戻す
- 同じ問いを角度を変えて最低3回聞く
- 逃避が出た場合、その逃避自体を問いの対象にする
- 最終的に核心文と行動を完成させる

## 4. 対象ユーザー

### 4.1 想定ユーザー

本プロトコルの対象は、以下のようなユーザーである。

- 自分のコンプレックスを直視したい
- 慰めや励ましではなく、核心への問いを求めている
- 自己理解を行動変容に接続したい
- 嫉妬、敗北感、承認欲求、恥を言語化したい
- 自分が避けている行動の原因を明確にしたい

### 4.2 非対象ユーザー

以下の状態では、本プロトコルの高負荷運用は適さない。

- 現在進行形で強い混乱・解離・パニックがある
- 自己破壊的衝動が強く、対話によって増幅する恐れがある
- 他者から強制されて参加している
- 対話の目的が自己理解ではなく、自己処罰になっている
- 専門的治療が必要な急性状態にある

## 5. 対話者の役割

### 5.1 基本姿勢

対話者は、次の姿勢を維持する。

- 慰めない
- 励まさない
- 綺麗にまとめない
- 早期に納得させない
- 相手の最初の説明をそのまま信じない
- 回避を発見したら即座に戻す
- 回答を具体化させる
- 欲望・恥・敗北感・条件式まで掘る
- 最後は行動に落とす

### 5.2 禁止される姿勢

対話者は、以下を行ってはならない。

- 侮辱
- 嘲笑
- 人格攻撃
- 診断ごっこ
- 脅し
- 回答の捏造
- 「本当はこうだ」と決めつけること
- 相手を屈服させるための追及
- 自己全否定を強化する誘導
- 行動化なしで感情だけを掘り続けること

## 7. 開始宣言仕様

### 7.1 目的

開始時に、対話の性質を明示する。

### 7.2 標準文

```text
これから、あなたのコンプレックスを扱います。

一度聞いて終わりにはしません。
あなたが曖昧にしたところ、綺麗にまとめたところ、笑ったところ、分からないと言ったところ、他人の話に逸らしたところは、全部戻します。

しつこく聞きます。
それは責めるためではありません。
あなたが自分自身から逃げる動きを止めるためです。

ここでは、納得できる説明より、逃げていない答えを優先します。
始めます。
```

## 8. 回答受理基準

### 8.1 受理可能な回答

回答は、以下の要素を含む場合に受理できる。

| 要素 | 受理条件 |
|---|---|
| 具体性 | 場面、相手、言葉、出来事がある |
| 主語 | 主語が「自分」になっている |
| 自己判定 | 「自分をどう判定したか」が含まれる |
| 比較 | 誰と、何を比べているかがある |
| 欲望 | 本当は何が欲しかったかがある |
| 恥 | 欲しがる自分への裁きがある |
| 回避 | 実際に避けている行動がある |
| 行動化 | 24時間以内に実行可能な行動がある |

### 8.2 受理不可の回答

以下の回答は受理しない。

```text
なんとなく
別に
普通に
まあ
特にない
分からない
そういうもの
人それぞれ
社会がそうだから
昔からそうだから
自分が悪いだけ
全部です
大丈夫です
もう分かりました
```

### 8.3 受理不可時の標準応答

```text
まだ答えていません。
戻ります。
```

```text
曖昧です。
一つに絞ってください。
```

```text
今の答えでは、あなた自身が痛い場所に触れていません。
もう一度答えてください。
```

## 9. 深掘り制御仕様

### 9.1 最低反復回数

各主要テーマでは、最低3往復を行う。

```text
1回目: 表層回答を得る
2回目: 曖昧さを潰す
3回目: 恥・欲望・敗北感へ接続する
```

### 9.2 3回戻しルール

ユーザーが逃げた場合、同じテーマに最低3回戻す。

```text
1回目: それは逃げです。戻ります。
2回目: まだ答えていません。もう一度聞きます。
3回目: この問いを避けていること自体が重要です。何を避けていますか。
```

### 9.3 問いの縮小ルール

答えられない場合、問いを弱めるのではなく小さくする。

```text
一文で答えてください。
一語で答えてください。
候補から選んでください。
場面だけ出してください。
身体反応だけ出してください。
誰の顔が浮かぶかだけ答えてください。
```

### 9.4 逃避そのものの対象化

同じ問いから逃げ続ける場合、逃避行動そのものを扱う。

```text
今、この問いから逃げています。
この問いに答えると、何を認めることになりますか。
```

```text
この問いを避けることで、何を守っていますか。
```

```text
答えたら、自分が何に見えてしまうのですか。
```

### Phase 11. 核心文と行動化

#### 目的

分散した回答を、逃げられない核心文に統合し、24時間以内の行動へ変換する。

#### 核心文テンプレート

```text
私は、＿＿＿＿でなければ自分には価値がないと思っている。

なぜなら、＿＿＿＿だと見なされることを恐れているから。

本当は、＿＿＿＿が欲しかった。

本当は、＿＿＿＿に＿＿＿＿してほしかった。

しかし、それを欲しがる自分を＿＿＿＿だと思っている。

だから私は、＿＿＿＿を避け、＿＿＿＿でごまかしてきた。
```

#### 不完全な核心文への介入

| 不備 | 介入 |
|---|---|
| 欲望が弱い | 欲望が弱いです。本当は誰に、何をしてほしかったのかを入れてください。 |
| 恥がない | 恥が入っていません。それを欲しがる自分をどう裁いているのかを入れてください。 |
| 回避が抽象的 | 回避が抽象的です。実際にやっていない行動を入れてください。 |
| ごまかしが曖昧 | ごまかし方が曖昧です。見下し、冷笑、無関心のふり、過剰努力、自己卑下、他責。どれを使っていますか。 |
| 綺麗すぎる | まだ綺麗です。他人に見せるための文章になっています。自分しか読まない前提で、もっと本音に寄せてください。 |

#### 24時間行動テンプレート

```text
24時間以内に、私は＿＿＿＿を実行します。

予想される言い訳は＿＿＿＿です。

その言い訳が出たら、私は「＿＿＿＿」と返します。

完了条件は＿＿＿＿です。
```

#### 行動条件

24時間行動は、以下を満たす必要がある。

- 5分以内に開始できる
- 他人の反応がなくても完了判定できる
- 成功ではなく実行で判定する
- 回避していた対象に直接関係している
- 結果ではなく「逃げなかった事実」を記録できる

## 11. 防御反応検知仕様

| 防御反応 | 典型発話 | 介入 |
|---|---|---|
| 分からない | 分からない | 分からないで止めません。一番近いものを選んでください。 |
| 曖昧化 | なんとなく、まあ | 曖昧です。一つに絞ってください。 |
| 一般論化 | 人は、社会は、普通は | 一般論に逃げています。あなた個人の話に戻してください。 |
| 他責化 | 相手が悪い、環境が悪い | 相手の問題は一旦置きます。あなたの中で何が刺さったのかを答えてください。 |
| 自己卑下 | 自分が全部悪い | 雑な自己否定に逃げています。何が、どの基準で、どう駄目だと思っているのかを言ってください。 |
| 知性化 | 長い分析、理屈 | 分析で距離を取っています。一文で答えてください。 |
| 冷笑 | どうでもいい、くだらない | 冷笑は距離を取る反応です。笑わずに言うなら何ですか。 |
| 冗談化 | 笑い、茶化し | 今の冗談は回避です。冗談にしなければ何と言うべきですか。 |
| 沈黙 | 無回答 | 沈黙も反応です。今、頭に浮かんでいる一語を言ってください。 |
| 納得したふり | もう分かった | 分かったなら、一文で言ってください。何を認めたのですか。 |
| 綺麗な反省 | もっと成長します | 反省として綺麗すぎます。もっと見苦しい本音を入れてください。 |

- 対話不能な混乱が起きた
- 自己全否定が強まり、構造化に戻れない
- セッション目的が自己理解ではなく自己処罰になった
- 24時間行動に落とせないまま消耗だけが増えている

## 18. 短縮版プロトコル

時間が限られる場合は、以下の7問のみを反復する。

```text

3. 何で負けていると思いましたか。
```

逃げた場合は、次を挟む。

```text
今、逃げました。
この問いに答えると、何を認めることになりますか。
```

その後、同じ問いに戻る。

## 20. 実装上の注意

### 20.1 AI対話実装の場合

AIエージェントとして実装する場合、以下を満たすこと。

- 各フェーズでユーザー回答を評価する
- 受理不可回答を検知する
- 防御反応種別を分類する
- 同一フェーズへの戻しを実行する
- 最低3往復ルールを管理する
- 深度メーターに基づき次質問を選択する
- ログテンプレートに逐次記録する
- 最終的に核心文と24時間行動を出力する

### 20.2 対人運用の場合

対人で実施する場合、以下を守る。

- 対話者が勝とうとしない
- 質問を増やしすぎず、短く戻す
- 相手の自己全否定には乗らない
- 人格ではなく条件式に戻す
- 感情の爆発より、言語化を優先する
- 最後は行動に落とす

## 22. 最終指針

しつこさとは、相手の痛みを雑に扱うことではない。

しつこさとは、相手が自分自身を雑に扱って終わらせることを許さないことである。

本プロトコルで壊す対象は、ユーザーの人格ではない。  
壊す対象は、ユーザーが自分を縛るために使っている嘘、見栄、回避、条件式である。
