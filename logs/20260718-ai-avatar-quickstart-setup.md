# AIアバター 導入手順書（最速立ち上げ構成）

## この手順書の前提（オーナー確認事項）
- 構成は前回の「バランス／最速」プリセットを採用。**ARDY（全身AIモーション）は使わず**、まず配信を成立させる
- モデル型: **3D VRM**（オリジナルキャラをアニメ調3Dで用意）
- 会話基盤: **AITuberKit**（Pixiv公開 ChatVRM のフォーク・OSS）で会話・コメント連動・VRM表示・口パクを一括で担う
- LLM: **Claude（Anthropic）** を想定（他社LLMにも差し替え可）
- 配信先: **YouTube Live**、映像合成は **OBS Studio**
- 上記が方針と異なる場合はここだけ差し替えれば、以降の手順はほぼ流用できる

### なぜこの構成が最小工数か
AITuberKitは、VRMをブラウザ内で表示し、**モーションタグによる身振り制御・音声からの口パク・YouTubeコメント連動・アイドル発話**までを内蔵している。そのため「②動かす・③会話・④配信の大部分」を1アプリに集約でき、前回課題だった「ARDYスケルトン→VRMリターゲット」という難所を初手では回避できる。演者も不要。

### 全体フロー
```
[VRoidでVRM作成] → [AITuberKitにVRM読込＋LLM/TTS設定] 
      → [YouTubeコメント連動＋アイドル発話] 
      → [OBSのブラウザソースで取り込み] → [YouTube Live配信]
```

## 事前に用意するもの
- PC（Windows または macOS）
- **Node.js 24.x** / **npm 11.6.2 以上**（AITuberKitの動作要件）
- Git
- **Claude(Anthropic) APIキー**（LLM用）
- **音声合成(TTS)**: 日本語なら **VOICEVOX** または **AivisSpeech**（無償・ローカル）が導入容易。クラウドなら ElevenLabs 等
- **YouTubeチャンネル**（ライブ配信有効化済み）と、コメント取得用に **YouTube Data API キー** もしくは **OneComme（わんコメ）**
- **OBS Studio**（無償）

> ライセンス重要事項は末尾「ライセンス・セキュリティ確認事項」を必ず確認（特に商用利用）。

## Phase 1: オリジナルキャラのVRM作成（VRoid Studio）

### 手順
1. VRoid Studio（無償）をインストールし起動
2. 新規作成 → テンプレートをベースに、原案イラスト/設定資料を見ながら顔・髪型・体型・衣装・色を寄せる
3. 表情（喜怒哀楽）とまばたきのプリセットが有効か確認
4. 「エクスポート」→ **VRM形式**で書き出し（VRM 1.0 推奨。用途に応じ0.xも可）。ポリゴン/テクスチャは配信負荷に合わせて調整

### 完了判定（テスト）
- VRMファイルが書き出せる
- 必須ボーン（腰・背骨・首・頭・肩・腕・手・脚・足）が破綻せずTポーズで表示される
- まばたき・口の開閉ブレンドシェイプが存在する（口パク用）

## Phase 2: AITuberKit導入とキャラ設定

### 2-1. インストール
ターミナルで以下を順に実行:
```
git clone https://github.com/tegnike/aituber-kit.git
cd aituber-kit
npm install
cp .env.example .env
npm run dev
```
ブラウザで `http://localhost:3000` を開く。
（初回セットアップ後は Windows: `LAUNCH.bat` / macOS: `LAUNCH.command` のダブルクリックでも起動可。Docker運用なら `docker compose up -d`）

### 2-2. LLM（Claude）設定
- `.env` もしくは画面の設定で **Anthropic** を選択し、APIキーを登録
- AITuberKitはOpenAI/Anthropic/Google Gemini/Azure/Groq/Mistral/DeepSeek/OpenRouter 等に対応。将来の差し替えも容易

### 2-3. 音声合成（TTS）設定
- VOICEVOX または AivisSpeech をローカル起動し、AITuberKitのTTS設定で該当エンジンを選択
- キャラのイメージに合う話者/スタイルを選ぶ

### 2-4. キャラクター設定
- Phase 1のVRMを読み込む（3Dモデル指定）
- **システムプロンプト**でキャラの人格・口調・NG事項・世界観を記述（配信の"中身"はここで決まる）
- 身振りは**モーションタグ**でVRMに指示できる（全身モーキャプ不要）

### 完了判定（テスト）
- テキストを入力→Claudeが応答し、TTSで発話、口が動く
- 応答時の身振り・表情が破綻しない
- 会話履歴が短期記憶として保持される

---

## Phase 3: 配信接続（コメント連動＋OBS）

### 3-1. YouTubeコメント連動
- コメント取得元を **YouTube API** または **OneComme** から選択して設定
- 取得間隔・表示名などを調整
- 安全のため、NGワード/入力長制限（デモ端末モードの機能）や、拾うコメントの条件を設定

### 3-2. アイドリング（無コメント時）
- **会話継続モード**を有効化し、コメントが無い間もキャラが自発発話するようにする
- アイドル発話の元ネタ（固定フレーズ／時間帯あいさつ／AI生成）を選ぶ

### 3-3. OBSで取り込み・配信
1. OBSに**ブラウザソース**を追加し、URLに `http://localhost:3000` を指定（またはブラウザ全画面を**ウィンドウキャプチャ**）
2. 背景・オーバーレイ・コメント表示などを合成
3. YouTube Liveのストリームキーを設定し、配信開始

### 完了判定（テスト）
- 実際のYouTubeコメントに数秒以内で反応して発話する
- 無コメント時もアイドル発話で沈黙しない
- OBSのプレビューにアバターが正しく映り、音ズレがない
- NGワード等のフィルタが機能する

---

## Phase 4: 運用・全体テスト
- **通し配信テスト**: 30〜60分の連続稼働で、遅延増大・メモリ肥大・音ズレ・クラッシュがないか
- **フェイルセーフ**: LLM/TTSがエラー時にキャラが黙り込まない挙動（アイドル発話へフォールバック）を確認
- **再現性**: `.env`（鍵はマスク）・使用モデル話者・OBSシーン構成・起動手順を社内ドキュメント化。次回はLAUNCHスクリプトで再起動

## ⚠️ ライセンス・セキュリティ確認事項（商用前に必須）

- **AITuberKitは v2.0.0 以降カスタムライセンス**。個人・教育・非営利は無償だが、**商用利用には別途の商用ライセンスが必要**。収益化配信・法人利用の場合は取得手続きを行うこと（公式のライセンス案内を参照）
- **VRM/Live2Dモデルの利用規約**、**ロゴ利用規約**も別途確認
- **TTSの利用条件**: VOICEVOX等は多くがクレジット表記を要求。採用エンジンごとに商用条件を確認
- **APIキー管理**: Web公開時はクライアントから直接APIを叩かせない構成にする。公開APIルートでサーバー側シークレット（`OPENAI_API_KEY` 等）を使う場合、`AITUBERKIT_SERVER_SECRET_ACCESS_MODE` 等の設定が必要（未設定だと一部API が403）。本番はバックエンドで鍵を管理し、認証・アクセス制限を実装
- **各LLMの商用利用規約**（Anthropic等）にも準拠

## 発展フェーズ（後から差別化）

初手構成が安定したら、必要に応じて上位化する:
- **表情品質**: NVIDIA **Audio2Face**（無償・音声→表情/口パク）で口パク・表情を高品位化
- **全身AIモーション（ARDY）**: AITuberKitの **外部連携モード（WebSocket）** を接点に、ARDY推論→VRMリターゲット結果を流し込む構成を検証。前回ロードマップのPhase 3（スケルトン→VRMリターゲット）をスパイクしてGo/No-Go判断
- **配信ランタイム強化**: 演出を作り込むなら Warudo/VNyan/自作(Unity+UniVRM, three-vrm)へ表示層を移行
- **音声対話**: OpenAI Realtime API 等で低遅延の音声対話モードを追加
