# 議事録

**議事録タイトル**  
PrismML Bonsai 1.7B（1-bit LLM）の性能・ユースケース・実装例・類似モデル比較に関する情報共有・議論  

**日時**  
2026年4月17日  

**議題**  
本件はX投稿（https://x.com/ErickSky/status/2044558671384822130）を起点とし、Bonsai 1.7Bの詳細性能、ユースケース、具体的な実装例、類似軽量LLMとの比較について徹底的に調査・整理した内容を、意思決定プロセスへ引き継ぐための資料としてまとめる。  

**議事内容**  

### 1. 導入・背景  
X投稿にてPrismMLのBonsai 1.7Bが紹介された。  
- 1.7Bパラメータを1-bit量子化で290MB程度に圧縮  
- ブラウザのWebGPUで100 tokens/secを実現  
- 従来の数GB規模モデルとは異なり、古いノートPCやスマートフォンでも即時動作可能  
- インターネット不要・完全プライベートなローカルAIとして、普及の転換点になるとの主張  
本件はこれを起点に、性能の詳細確認、実際のユースケース、実装可能性、競合比較まで深掘りした。  

### 2. Bonsai 1.7Bの性能詳細  
- **基本スペック**  
  - パラメータ数：1.7B（約1.4B non-embedding）  
  - アーキテクチャ：Qwen3-1.7Bベース（denseモデル、GQA: 16 query / 8 KV heads、SwiGLU MLP、RoPE、RMSNorm、28層）  
  - コンテキスト長：32,768 tokens  
  - ボキャブラリサイズ：151,936  
  - 重み形式：1-bit（end-to-end量子化、embeddings・attention projections・MLP projections・LM headすべて1-bit）  
  - モデルファイル形式：GGUF Q1_0 (g128)形式、グループごとにFP16スケール共有（実効約1.125 bits/weight）  
  - モデルサイズ（展開時）：約0.24-0.29 GB（FP16版の約14倍圧縮）  
  - ライセンス：Apache 2.0  

- **メモリ使用量と効率**  
  極めて軽量で、スマホ・古いノートPC・ブラウザ（WebGPU）で動作可能。通常の同規模モデル（FP16）は約3.4 GB必要だが、大幅に下回る。  

- **推論速度（Throughput）の例**  
  - iPhone 17 Pro Max：約130 tokens/sec（生成）  
  - M4 Pro Mac：高スループット（4Bモデルで132 t/sの例あり、1.7Bも同等レベル）  
  - RTX 4090 (llama.cpp CUDA)：674 tokens/sec (TG128) — FP16比約3.0倍高速  
  - M4 Pro (Metal)：250 tokens/sec (TG128) — FP16比3.8倍  
  - ブラウザ (WebGPU)：約100 tokens/sec前後  
  エネルギー効率も高く、エッジデバイスで75-80%削減の可能性あり。  

- **ベンチマーク性能**  
  1.7B特有の詳細スコアは公開情報が限定的だが、同サイズクラスでleading accuracyと評価。  
  - 8B版の参考平均スコア（MMLU-Redux, MuSR, GSM8K, HumanEval+, IFEval, BFCLv3など）：約70.5  
  - 1.7Bはサイズに応じてスケールダウンしつつ、同規模（1-3Bクラス）のフル精度モデルを上回るか競争力あり。特に数学（GSM8K）や指示追従で強い傾向。  
  複雑な推論タスクでは8Bに劣るが、基本会話・分類・要約・シンプルな数学などで実用的。  
  強み：サイズ vs 性能のParetoフロンティアを大幅に左シフト。  
  注意点：1-bit圧縮のトレードオフで、非常に高度な推論や長文生成ではフル精度大規模モデルに劣る場合あり。  

### 3. 主なユースケース  
Bonsai 1.7Bの最大の強みはサイズ240-290MBとブラウザ/スマホ即時動作（iPhoneで100-130 tokens/sec前後）。フル精度の大規模モデルに比べて高度な推論は劣るが、エッジ/ローカル実行に最適。  

- **1. モバイル・エッジデバイスでの日常AIアシスタント**  
  スマホ・タブレット・ウェアラブルでオフライン動作。リアルタイムチャット、メール/メッセージの下書き、ToDoリスト作成、簡単な質問応答。インターネット不要、プライバシー保護、低消費電力。  

- **2. テキスト処理・要約**  
  長い文章の要約、分類、キーポイント抽出。会議メモや記事の要約、SNS投稿の下書き、メール作成で特に優秀と評価。  

- **3. ブラウザ内即時AI（インストール不要）**  
  WebGPUデモでブラウザだけで動作。ウェブページ閲覧中にリアルタイム説明・翻訳・アイデア生成。教育用途にも有効。  

- **4. シンプルなコーディング支援・スクリプト生成**  
  基本的なPythonスクリプト、コマンドラインゲーム、簡単な計算ツール作成。数値計算、データ処理スクリプト、LaTeX数式生成。  

- **5. プライバシー重視のエンタープライズ/個人利用**  
  機密データをクラウドに送らずローカル処理。オフラインAIエージェント、個人知識ベースとのチャット、リアルタイムロボティクス制御。  

- **6. その他のニッチユースケース**  
  教育・学習ツール（シンプル数学問題解説、語学練習）、コンテンツ生成（短編ストーリー、アイデアブレインストーム）、組み込み/リアルタイム（低電力デバイスでの軽量推論）。  

強い点：基本会話、要約、指示追従、シンプル数学・論理。  
弱い点：高度なコーディング、複雑推論、長文生成での幻覚（小規模モデル共通）。  

### 4. 具体的な実装例  
Bonsai 1.7Bはブラウザ・スマホ・PCで簡単に実装可能。主な方法を以下に記載（コードは省略せず全文記載）。  

**4-1. 一番簡単：ブラウザ（WebGPU）で即時試す（インストール不要）**  
デモURL: https://huggingface.co/spaces/webml-community/bonsai-webgpu  
手順：  
1. リンクを開く（Chrome/Edge推奨）。  
2. Bonsai 1.7Bを選択。  
3. 初回にモデル（約290MB）をダウンロード。  
4. チャット開始（100 tokens/sec前後）。  

**4-2. Mac（Apple Silicon）でMLXを使う**  
```python
# インストール（PrismML forkが必要）
pip install mlx-lm
pip install mlx @ git+https://github.com/PrismML-Eng/mlx.git@prism

# 実行例
from mlx_lm import load, generate

model, tokenizer = load("prism-ml/Bonsai-1.7B-mlx-1bit")

response = generate(
    model,
    tokenizer,
    prompt="量子コンピューティングを小学生向けに説明して。",
    max_tokens=512,
    temp=0.7
)
print(response)
```
iPhone/iPad：MLX Swift（PrismML fork）でアプリ化可能。速度例：iPhone 17 Pro Maxで100-130 tokens/sec。  

**4-3. llama.cpp（Windows/Linux/Mac/CPU/GPU対応）**  
PrismMLのfork版を使用。  
```bash
git clone https://github.com/PrismML-Eng/llama.cpp.git
cd llama.cpp
cmake -B build -DGGML_METAL=ON -DGGML_CUDA=ON  # 必要に応じて
cmake --build build --config Release -j

# モデルダウンロード
wget https://huggingface.co/prism-ml/Bonsai-1.7B-gguf/resolve/main/Bonsai-1.7B.gguf

# 実行例（CLI）
./build/bin/llama-cli \
  -m Bonsai-1.7B.gguf \
  -p "Pythonで簡単なToDoアプリのコードを書いて" \
  -n 512 --temp 0.8 -ngl 99  # GPU offload
```

**サーバー起動例（OpenAI互換API）**  
```bash
./build/bin/llama-server -m Bonsai-1.7B.gguf --port 8080 --host 0.0.0.0
```

**4-4. 実用的な統合例（Pythonアプリ）**  
```python
# GradioやStreamlitでシンプルUI
import gradio as gr
from mlx_lm import load, generate  # または llama-cpp-python

model, tokenizer = load("prism-ml/Bonsai-1.7B-mlx-1bit")

def chat(message, history):
    response = generate(model, tokenizer, prompt=message, max_tokens=256)
    return response

gr.ChatInterface(chat).launch()
```
拡張：LangChainでRAG、AutoGenでエージェント、ロボット制御などに組み込み可能。  

**その他**  
- DockerやGoogle Colabでも即時起動可能。  
- 詳細ドキュメント：PrismML Bonsai-demo GitHub（https://github.com/PrismML-Eng/Bonsai-demo/）およびHugging Faceモデルカード。  
- 注意点：コンテキスト32k tokens対応。Ternary（1.58-bit）版も0.37GBで利用可能（性能向上）。  

### 5. 類似軽量LLMとの比較  
（2026年4月時点、主に500MB以下の超軽量1-3Bクラスモデル中心）  

| モデル                  | パラメータ | サイズ (quant) | 速度例 (tokens/sec)          | 性能目安 (平均ベンチ) | 強み・弱み                          | ライセンス |
|-------------------------|------------|----------------|------------------------------|-----------------------|-------------------------------------|------------|
| **Bonsai 1.7B** (1-bit) | 1.7B      | **0.24-0.29GB** | iPhone: 100-130<br>RTX4090: 674<br>ブラウザ: ~100 | 同規模トップクラス（8B版70.5参考）<br>要約・指示追従良好 | **最小サイズ・最速エッジ**<br>1-bit全層圧縮、ブラウザ最適 | Apache 2.0 |
| **Qwen3-0.6B / 1.7B** (Q4/Q5) | 0.6-1.7B  | 0.4-0.9GB     | スマホ: 50-100<br>GPU: 高   | MMLU ~60-68<br>GSM8K良好 | バランス良く高性能、多言語強い | Apache 2.0 |
| **Llama 3.2 1B** (INT4) | 1B        | ~0.74GB       | ブラウザ/GPU: 8-15          | 指示品質良好         | ブラウザ対応良いがダウンロード大 | Meta |
| **TinyLlama 1.1B** (INT4) | 1.1B     | ~0.64GB       | 低スペック: 10-18           | 基本タスクのみ       | 超軽量だが性能低め | Apache 2.0 |
| **Phi-3/4 Mini** (~3-4B) | 3.8B     | 2GB+ (quant)  | 中スペック: 中              | 推論・多言語強い     | 高品質だが大幅に重い | MIT       |
| **Gemma 2/3 小規模** (E2Bなど) | 2B前後   | 1GB前後       | 変動                    | 競争力あり           | Google系、モバイル向け | 制限あり  |

**比較ポイント**  
- **サイズ・メモリ効率**：Bonsaiが圧倒的（FP16比14x圧縮）。231-290MBでiPhoneや古いPCでも動作。  
- **速度**：Bonsaiがエッジで突出（1-bit加算中心演算が有利）。  
- **性能**：同サイズクラスでleading。要約・基本会話・シンプル数学で実用的。複雑推論は小規模共通の限界あり。  
- **対応プラットフォーム**：BonsaiはWebGPUブラウザ即時、llama.cpp fork、MLX、iOS対応が抜群。  

**おすすめの選び方**  
- 最軽量・オフライン最優先 → Bonsai 1.7B  
- バランス性能重視 → Qwen3-1.7B/4B（quant）  
- 指示・推論品質 → Phi MiniやLlama 3.2 1B
