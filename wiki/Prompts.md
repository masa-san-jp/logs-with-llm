# プロンプト集 (Prompts)

`prompts/` ディレクトリに再利用可能なプロンプトを保存しています。

## ファイル一覧

| ファイル | 内容 |
|---|---|
| `20260120-writer-voice.yml` | 文章スタイル・声のトーン定義 |
| `20260218-v1.0` | 議事録作成プロンプト v1.0 |
| `20260218-v1.1` | 議事録作成プロンプト v1.1 |

## 議事録作成プロンプト

### 基本版

```
意思決定のプロセスを記録しておくために、このチャットの会話の議事録を作って、
マークダウン形式でmdファイルに書き出して。
最終成果物は別のmdファイルで書き出して。
```

### 使い方

1. LLMとの会話が一段落したら、上のプロンプトを送る
2. LLMが2つのファイル（議事録と成果物）を出力する
3. `logs/YYYYMMDD-{llm名}-{テーマ}.md` として保存する

## 会話継続プロンプト

過去の議事録を使って会話を続けるときのテンプレートです。

### 同じLLMで続ける

```
（前回の議事録をここに貼る）

---

この話の続きなんだけど、〇〇について詳しく考えたい。
```

### 別のLLMに持ち込む

```
（前回の議事録をここに貼る）

---

別のAIとして、この設計についてどう思う？
異なる視点や改善案があれば教えて。
```

### 複数LLMで討論する

```
（議事録A・議事録Bをまとめて貼る）

---

ClaudeとGeminiがそれぞれ別の結論に達しています。
あなたはどちらの意見に同意しますか？また、第3の視点はありますか？
```

## プロンプトの追加方法

新しいプロンプトを追加する場合:

1. `prompts/YYYYMMDD-{名前}.yml` または `.md` で作成
2. YAML形式の場合の基本構成:

```yaml
name: プロンプト名
version: "1.0"
description: このプロンプトの説明
prompt: |
  ここにプロンプトの内容を書く
tags:
  - タグ1
  - タグ2
```

## ライタースタイル定義 (writer-voice.yml)

`prompts/20260120-writer-voice.yml` にはブログ記事の文体・声のトーンが定義されています。週次ブログ自動化スクリプトがこれを参照して、一貫したスタイルで記事を生成します。
